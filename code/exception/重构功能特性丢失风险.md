### 2.1 🔴 高风险特性

#### 2.1.1 负载均衡器管理

#### 2.1.3 双 CA 模式

```go
// 双 CA 模式：支持自签名 CA（默认）和用户自定义 Global CA
func (c *CertsGenerator) LookUpOrGenerate() error {
    // 1. 检查是否使用用户自定义 CA
    isUserCustomCA := c.checkUserCustomCA()
    
    // 2. 生成或查找 CA 证书
    if isUserCustomCA {
        // 使用用户自定义 Global CA
        caCert = c.getUserCustomCA()
    } else {
        // 生成自签名 CA
        caCert = c.generateSelfSignedCA()
    }
    
    // 3. 使用 CA 签发其他证书
    c.signCertificates(caCert)
}
```

```go
// 方案：在创建 KubeadmConfigTemplate 时，从 ConfigMap 读取并注入
func (e *EnsureClusterAPIObj) createKubeadmConfigTemplate() error {
    // 1. 从 ConfigMap 读取自定义脚本
    configMap := e.getPreprocessConfigMap()
    scripts := e.parseScriptsFromConfigMap(configMap)
    
    // 2. 构建 preKubeadmCommands
    preCommands := []string{}
    for _, script := range scripts {
        if script.Phase == "pre" {
            preCommands = append(preCommands, script.Content)
        }
    }
    
    // 3. 构建 postKubeadmCommands
    postCommands := []string{}
    for _, script := range scripts {
        if script.Phase == "post" {
            postCommands = append(postCommands, script.Content)
        }
    }
    
    // 4. 创建 KubeadmConfigTemplate
    template := &bootstrapv1.KubeadmConfigTemplate{
        Spec: bootstrapv1.KubeadmConfigTemplateSpec{
            Template: bootstrapv1.KubeadmConfigTemplateResource{
                Spec: bootstrapv1.KubeadmConfigSpec{
                    PreKubeadmCommands:  preCommands,
                    PostKubeadmCommands: postCommands,
                },
            },
        },
    }
}
```

#### 2.2.1 证书 SANs 注入

**当前实现**:
```go
// 节点感知的 SANs 注入：根据 BKENode 列表动态计算
func (c *CertsGenerator) generateAPIServerCert() error {
    // 1. 获取所有 Master 节点 IP
    masterNodes := c.getMasterNodes()
    
    // 2. 构建 SANs
    sans := []string{
        "127.0.0.1",
        "localhost",
        c.Cluster.Spec.ControlPlaneEndpoint.Host,  // VIP

    for _, node := range masterNodes {
        sans = append(sans, node.IP)
        sans = append(sans, node.Hostname)

    sans = append(sans, c.Cluster.Spec.ClusterConfig.ControlPlane.APIServer.CertSANs...)
```

## 3. 风险汇总与缓解策略

### 3.1 风险汇总表

| 风险等级 | 功能特性数量 | 占比 | 关键特性 |
|---------|------------|------|---------|
| 🔴 **高** | 8 | 40% | 负载均衡器、节点环境初始化、双 CA 模式、自定义脚本 |
| 🟡 **中** | 4 | 20% | 证书 SANs 注入、HA KubeConfig、etcd SANs、API Server SANs |
| 🟢 **低** | 8 | 40% | 标准 Kubernetes 配置 |
| **总计** | 20 | 100% | - |

### 3.2 推荐的缓解策略

#### 策略一：保留关键 Phase，渐进式重构

```
重构前 Phase 顺序：
1. EnsureBKEAgent
2. EnsureNodesEnv
3. EnsureClusterAPIObj
4. EnsureCerts
5. EnsureLoadBalance
6. EnsureMasterInit
7. EnsureMasterJoin
8. EnsureWorkerJoin
9. EnsureAddonDeploy
10. EnsureNodesPostProcess

重构后 Phase 顺序：
1. EnsureBKEAgent（保留）
2. EnsureNodesEnv（保留）← 节点环境初始化
3. EnsureCerts（保留）← 双 CA 模式、节点感知 SANs
4. EnsureLoadBalance（保留）← HAProxy + Keepalived
5. EnsureClusterAPIObj（重构）← 创建 KCP、KCT
6. EnsureControlPlaneReady（新增）← 等待 KCP 就绪
7. EnsureWorkerJoin（重构）← 使用 KCT
8. EnsureAddonDeploy（保留）

```

**优点**:
- ✅ 保留所有 BKE 特有功能
- ✅ 渐进式重构，风险可控
- ✅ 易于回滚

**缺点**:
- ⚠️ Phase 数量较多
- ⚠️ 部分功能重复（如证书生成）

#### 策略二：完全使用 CAPI 标准，牺牲部分功能

```
重构后 Phase 顺序：
1. EnsureBKEAgent（保留）
2. EnsureClusterAPIObj（重构）← 创建 KCP、KCT
   • 在 preKubeadmCommands 中实现节点环境初始化
   • 在 files 中推送证书和配置
3. EnsureControlPlaneReady（新增）← 等待 KCP 就绪
4. EnsureWorkerJoin（重构）← 使用 KCT
5. EnsureAddonDeploy（保留）
```

**优点**:
- ✅ Phase 数量少
- ✅ 与 CAPI 标准完全一致
- ✅ 易于理解和维护

**缺点**:
- ❌ 双 CA 模式丢失
- ❌ 负载均衡器管理丢失（需要外部实现）
- ❌ 自定义脚本灵活性降低
- ❌ preKubeadmCommands 过于复杂

#### 策略三：混合模式（推荐）

```
重构后 Phase 顺序：
1. EnsureBKEAgent（保留）
2. EnsureNodesEnv（保留）← 节点环境初始化
3. EnsureCerts（保留）← 双 CA 模式、节点感知 SANs
4. EnsureLoadBalance（保留）← HAProxy + Keepalived
5. EnsureClusterAPIObj（重构）← 创建 KCP、KCT
   • 使用预生成的证书
   • 注入 SANs
6. EnsureControlPlaneReady（新增）← 等待 KCP 就绪
7. EnsureWorkerJoin（重构）← 使用 KCT
8. EnsureAddonDeploy（保留）
9. EnsureNodesPostProcess（重构）← 从 ConfigMap 读取并注入到 KCT
```

**优点**:
- ✅ 保留所有关键功能
- ✅ 利用 CAPI 标准能力
- ✅ Phase 数量适中
- ✅ 易于维护和扩展

**缺点**:
- ⚠️ 需要适配现有逻辑
- ⚠️ 部分功能需要重构

## 4. 最终建议

### 4.1 推荐方案：混合模式（策略三）

**理由**:
1. **保留关键功能**: 双 CA 模式、负载均衡器、节点环境初始化等 BKE 特有功能得以保留
2. **利用 CAPI 标准**: 使用 KubeadmControlPlane 和 KubeadmConfigTemplate 的标准能力

## 5. 总结

### 5.1 风险评估

使用 KubeadmConfigTemplate 与 KubeadmControlPlane 重构后：

| 评估维度 | 结论 |
|---------|------|
| **功能特性丢失风险** | 🔴 **中高风险**（40% 高风险特性） |
| **规格丢失风险** | 🟡 **中风险**（部分 BKE 特有规格需要适配） |
| **兼容性风险** | 🟢 **低风险**（标准 Kubernetes 配置完全兼容） |
| **维护性风险** | 🟡 **中风险**（需要适配现有逻辑） |

### 5.2 关键结论

1. **不能完全依赖 CAPI 标准**:
   - BKE 有大量特有功能（双 CA 模式、负载均衡器、节点环境初始化等）
   - 这些功能在 CAPI 标准中不存在或不满足需求
2. **推荐混合模式**:
   - 保留关键 Phase（EnsureCerts、EnsureLoadBalance、EnsureNodesEnv）
   - 使用 CAPI 标准管理控制平面和 Worker 节点
   - 通过适配层连接两者
3. **需要额外开发**:
   - 证书注入机制（EnsureCerts → KCP）
   - 自定义脚本注入机制（EnsureNodesPostProcess → KCT）
   - HA KubeConfig 生成机制
4. **渐进式重构**:
   - 不要一次性移除所有 Phase
   - 逐步验证每个重构步骤
   - 保持回滚能力

### 5.3 最终建议
**采用混合模式（策略三）**，保留 BKE 的关键功能特性，同时利用 CAPI 标准能力，实现渐进式重构。这样可以：
- ✅ 保留所有 BKE 特有功能
- ✅ 利用 CAPI 标准能力
- ✅ 降低重构风险
- ✅ 易于维护和扩展

### 3.1 新增 EnsureControlPlaneReady Phase

#### 3.1.1 Phase 实现

```go
// pkg/phaseframe/phases/ensure_controlplane_ready.go
package phases

import (
    "context"
    "time"
    
    "github.com/pkg/errors"
    ctrl "sigs.k8s.io/controller-runtime"
    "sigs.k8s.io/controller-runtime/pkg/client"
    
    clusterv1 "sigs.k8s.io/cluster-api/api/v1beta1"
    controlplanev1 "sigs.k8s.io/cluster-api/controlplane/kubeadm/api/v1beta1"
    
    confv1beta1 "gopkg.openfuyao.cn/cluster-api-provider-bke/api/bkecommon/v1beta1"
    bkev1beta1 "gopkg.openfuyao.cn/cluster-api-provider-bke/api/capbke/v1beta1"
    "gopkg.openfuyao.cn/cluster-api-provider-bke/pkg/phaseframe"
    "gopkg.openfuyao.cn/cluster-api-provider-bke/pkg/phaseframe/phaseutil"
    "gopkg.openfuyao.cn/cluster-api-provider-bke/utils/capbke/constant"
    "gopkg.openfuyao.cn/cluster-api-provider-bke/utils/capbke/feature"
)

const (
    EnsureControlPlaneReadyName confv1beta1.BKEClusterPhase = "EnsureControlPlaneReady"
    ControlPlaneReadyTimeout    = 10 * time.Minute
    ControlPlaneReadyInterval   = 5 * time.Second
)

type EnsureControlPlaneReady struct {
    phaseframe.BasePhase
}

func NewEnsureControlPlaneReady(ctx *phaseframe.PhaseContext) phaseframe.Phase {
    base := phaseframe.NewBasePhase(ctx, EnsureControlPlaneReadyName)
    return &EnsureControlPlaneReady{BasePhase: base}
}

func (e *EnsureControlPlaneReady) Execute() (ctrl.Result, error) {
    // 检查 Feature Gate
    if !feature.IsFeatureEnabled(feature.KubeadmControlPlane) {
        // Feature 未启用，跳过此 Phase
        return ctrl.Result{}, nil
    }
    
    ctx, c, bkeCluster, _, log := e.Ctx.Untie()
    
    // 1. 获取 KubeadmControlPlane
    kcp, err := phaseutil.GetKubeadmControlPlane(ctx, c, bkeCluster)
    if err != nil {
        log.Error(constant.ControlPlaneNotReadyReason, "Failed to get KubeadmControlPlane: %v", err)
        return ctrl.Result{}, err
    }
    
    if kcp == nil {
        log.Info(constant.ControlPlaneNotReadyReason, "KubeadmControlPlane not found, waiting...")
        return ctrl.Result{RequeueAfter: ControlPlaneReadyInterval}, nil
    }
    
    // 2. 检查 KCP 是否就绪
    if !e.isKubeadmControlPlaneReady(kcp) {
        log.Info(constant.ControlPlaneNotReadyReason, 
            "KubeadmControlPlane not ready, status: %v", kcp.Status)
        return ctrl.Result{RequeueAfter: ControlPlaneReadyInterval}, nil
    }
    
    // 3. 检查所有控制平面节点是否就绪
    ready, err := e.areControlPlaneMachinesReady(ctx, c, bkeCluster, kcp)
    if err != nil {
        log.Error(constant.ControlPlaneNotReadyReason, "Failed to check control plane machines: %v", err)
        return ctrl.Result{}, err
    }
    
    if !ready {
        log.Info(constant.ControlPlaneNotReadyReason, "Waiting for control plane machines to be ready")
        return ctrl.Result{RequeueAfter: ControlPlaneReadyInterval}, nil
    }
    
    // 4. 检查集群是否可访问
    if err := e.checkClusterHealth(ctx, c, bkeCluster); err != nil {
        log.Info(constant.ControlPlaneNotReadyReason, "Cluster not healthy: %v", err)
        return ctrl.Result{RequeueAfter: ControlPlaneReadyInterval}, nil
    }
    
    log.Info(constant.ControlPlaneReadyReason, "Control plane is ready")
    return ctrl.Result{}, nil
}

func (e *EnsureControlPlaneReady) NeedExecute(old, new *bkev1beta1.BKECluster) bool {
    if !e.BasePhase.DefaultNeedExecute(old, new) {
        return false
    }
    
    // 只有启用了 KubeadmControlPlane Feature 才需要执行
    if !feature.IsFeatureEnabled(feature.KubeadmControlPlane) {
        return false
    }
    
    // 如果控制平面未就绪，需要执行
    if !conditions.IsTrue(new, clusterv1.ControlPlaneInitializedCondition) {
        e.SetStatus(bkev1beta1.PhaseWaiting)
```

```go
func (e *EnsureWorkerBootstrap) Execute() (ctrl.Result, error) {
    // 检查 Feature Gate
    if !feature.IsFeatureEnabled(feature.KubeadmConfigTemplate) {
        return ctrl.Result{}, nil
    }
    
    ctx, c, bkeCluster, _, log := e.Ctx.Untie()
    
    // 1. 获取 MachineDeployment
    md, err := phaseutil.GetMachineDeployment(ctx, c, bkeCluster)
    if err != nil {
        return ctrl.Result{}, err
    }
    
    // 2. 检查是否所有 Worker 都使用了 KubeadmConfigTemplate
    if !e.isUsingKubeadmConfigTemplate(md) {
        // 迁移到 KubeadmConfigTemplate
        if err := e.migrateToKubeadmConfigTemplate(ctx, c, bkeCluster, md); err != nil {
            return ctrl.Result{}, err
        }
    }

    return e.waitForWorkersReady(ctx, c, bkeCluster)
}
```

```go
    // 1. 获取 BKE 生成的证书 Secret
    certSecrets, err := i.getBKECertificateSecrets(ctx)

        nodeCertSecret := i.createNodeCertificateSecret(node, certSecrets)

                return err
            }
        }
    }
    
    return nil
```
```go
}

// Generate 生成 KubeadmConfigTemplate 配置
func (g *KCTGenerator) Generate(ctx context.Context) (*bootstrapv1.KubeadmConfigTemplate, error) {
    template := &bootstrapv1.KubeadmConfigTemplate{
        ObjectMeta: metav1.ObjectMeta{
            Name:      g.BKECluster.Name + "-worker-config",
            Namespace: g.BKECluster.Namespace,
            Labels: map[string]string{
                clusterv1.ClusterNameLabel: g.BKECluster.Name,
            },
            OwnerReferences: []metav1.OwnerReference{
                {
                    APIVersion: clusterv1.GroupVersion.String(),
                    Kind:       "Cluster",
                    Name:       g.BKECluster.Name,
                    UID:        g.BKECluster.UID,
                },
            },
        },
        Spec: bootstrapv1.KubeadmConfigTemplateSpec{
            Template: bootstrapv1.KubeadmConfigTemplateResource{
                Spec: g.generateKubeadmConfigSpec(),
            },
        },
    }
    
    return template, nil
}

// generateKubeadmConfigSpec 生成 KubeadmConfigSpec
func (g *KCTGenerator) generateKubeadmConfigSpec() bootstrapv1.KubeadmConfigSpec {
    spec := bootstrapv1.KubeadmConfigSpec{
        JoinConfiguration: g.generateJoinConfiguration(),
    }
    
    // 如果启用了 PreserveBKEFeatures，注入 BKE 特有配置
    if feature.IsFeatureEnabled(feature.PreserveBKEFeatures) {
        spec.Files = g.generateFiles()
        spec.PreKubeadmCommands = g.generatePreCommands()
        spec.PostKubeadmCommands = g.generatePostCommands()
    }
    
    return spec
}

// generateJoinConfiguration 生成 JoinConfiguration
func (g *KCTGenerator) generateJoinConfiguration() bootstrapv1.JoinConfiguration {
    return bootstrapv1.JoinConfiguration{
        NodeRegistration: bootstrapv1.NodeRegistration{
            Name:             "{{ .LocalHostname }}",
            CRISocket:        g.getCRISocket(),
            KubeletExtraArgs: g.getKubeletExtraArgs(),
        },
        Discovery: bootstrapv1.Discovery{
            BootstrapToken: &bootstrapv1.BootstrapTokenDiscovery{
                APIServerEndpoint: g.getControlPlaneEndpoint(),
                Token:             "",  // 由 CAPI 自动生成
                CACertHashes:      nil, // 由 CAPI 自动填充
            },
        },
    }
}

// generateFiles 生成文件列表
func (g *KCTGenerator) generateFiles() []bootstrapv1.File {
    files := []bootstrapv1.File{}
    
    // 1. Kubelet 配置文件
    files = append(files, bootstrapv1.File{
        Path:        "/etc/kubernetes/kubelet.conf",
        Content:     g.generateKubeletConfig(),
        Permissions: "0644",
    })
    
    // 2. Containerd 配置文件
    files = append(files, bootstrapv1.File{
        Path:        "/etc/containerd/config.toml",
        Content:     g.generateContainerdConfig(),
        Permissions: "0644",
    })
    
    // 3. 镜像仓库证书
    if g.Config.Cluster.ImageRepo.TlsSecretRef != nil {
        files = append(files, bootstrapv1.File{
            Path:        "/etc/openFuyao/certs/trust-chain.crt",
            Content:     g.getRegistryCert(),
            Permissions: "0644",
        })
    }
    
    // 4. 用户自定义文件
    if g.Config.Cluster.Files != nil {
        for _, f := range g.Config.Cluster.Files {
            files = append(files, bootstrapv1.File{
                Path:        f.Path,
                Content:     f.Content,
                Permissions: f.Permissions,
            })
        }
    }
    
    return files
}
```

```go
// executeWithKCT 使用 KCT 执行 Worker 节点加入
func (e *EnsureWorkerJoin) executeWithKCT() (ctrl.Result, error) {
    ctx, c, bkeCluster, cluster, log := e.Ctx.Untie()
    
    // 1. 检查控制平面是否已初始化
    if !conditions.IsTrue(cluster, clusterv1.ControlPlaneInitializedCondition) {
        log.Info("Control plane not initialized, waiting...")
        return ctrl.Result{RequeueAfter: 10 * time.Second}, nil
    }
    
    // 2. 获取期望的 Worker 节点数
    desiredWorkerCount := e.getDesiredWorkerCount()
    
    // 3. 获取 MachineDeployment
    md, err := phaseutil.GetMachineDeployment(ctx, c, cluster)
    if err != nil {
        return ctrl.Result{}, errors.Wrap(err, "failed to get MachineDeployment")
    }
    
    if md == nil {
        return ctrl.Result{}, errors.New("MachineDeployment not found")
    }
    
    // 4. 声明式更新 Replicas
    currentReplicas := int(*md.Spec.Replicas)
    if currentReplicas != desiredWorkerCount {
        log.Info("Scaling workers from %d to %d replicas", currentReplicas, desiredWorkerCount)
        
        md.Spec.Replicas = ptr.To[int32](int32(desiredWorkerCount))
        if err := c.Update(ctx, md); err != nil {
            return ctrl.Result{}, errors.Wrap(err, "failed to update MachineDeployment replicas")
        }
    }
    
    // 5. 等待所有 Worker 节点就绪
    return e.waitForWorkersReady(ctx, c, cluster, desiredWorkerCount)
}
```

```go
// ScriptInjector 自定义脚本注入器
type ScriptInjector struct {
    Client     client.Client
    BKECluster *bkev1beta1.BKECluster
}

// InjectScriptsToKCT 将自定义脚本注入到 KubeadmConfigTemplate
func (i *ScriptInjector) InjectScriptsToKCT(ctx context.Context, kct *bootstrapv1.KubeadmConfigTemplate) error {
    // 1. 从 ConfigMap 读取自定义脚本
    scripts, err := i.getCustomScripts(ctx)
    if err != nil {
        return err
    }
    
    // 2. 分类脚本
    preScripts := []string{}
    postScripts := []string{}
    
    for _, script := range scripts {
        if script.Phase == "pre" {
            preScripts = append(preScripts, script.Content)
        } else if script.Phase == "post" {
            postScripts = append(postScripts, script.Content)
        }
    }
    
    // 3. 更新 KCT
    kct.Spec.Template.Spec.PreKubeadmCommands = append(
        kct.Spec.Template.Spec.PreKubeadmCommands,
        preScripts...,
    )
    
    kct.Spec.Template.Spec.PostKubeadmCommands = append(
        kct.Spec.Template.Spec.PostKubeadmCommands,
        postScripts...,
    )
    
    return nil
}

// getCustomScripts 从 ConfigMap 获取自定义脚本
func (i *ScriptInjector) getCustomScripts(ctx context.Context) ([]CustomScript, error) {
    // 尝试获取 preprocess-all-config
    configMapName := "preprocess-all-config"
    configMap := &corev1.ConfigMap{}
    
    if err := i.Client.Get(ctx, client.ObjectKey{
        Namespace: i.BKECluster.Namespace,
        Name:      configMapName,
    }, configMap); err != nil {
        if apierrors.IsNotFound(err) {
            // 尝试获取 preprocess-config-batch-*
            return i.getBatchScripts(ctx)
        }
        return nil, err
    }
    
    // 解析脚本
    return i.parseScriptsFromConfigMap(configMap), nil
}
```

```go
// Execute 并行执行任务
func (e *ParallelExecutor) Execute(ctx context.Context, tasks []Task) error {
    g, ctx := errgroup.WithContext(ctx)
    g.SetLimit(e.concurrency)
    
    for _, task := range tasks {
        task := task
        g.Go(func() error {
            return task.Execute(ctx)
        })
    }
    
    return g.Wait()
```

- **可切换**：通过 Feature Gate 实现新旧模式的灵活切换

```
│  │  • 证书注入器（CertInjector）                       │   │
│  │  • 脚本注入器（ScriptInjector）                     │   │
│  │  • 配置生成器（KCPGenerator、KCTGenerator）         │   │
│  └─────────────────────────────────────────────────────┘   │
│                          ↓                                  │
│  Layer 2: CAPI 标准能力层（引入）                          │
│  ┌─────────────────────────────────────────────────────┐   │
│  │  • KubeadmControlPlane（控制平面管理）              │   │
│  │  • KubeadmConfigTemplate（Worker Bootstrap）        │   │
│  │  • MachineDeployment（Worker 管理）                 │   │
│  │  • Cluster API Controllers（生命周期管理）          │   │
│  └─────────────────────────────────────────────────────┘   │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```
**核心思路**：
- **Layer 0**：BKE 的核心竞争力，必须保留
- **Layer 1**：连接 Layer 0 和 Layer 2 的桥梁，负责数据转换和注入
- **Layer 2**：CAPI 标准能力，提供声明式 API 和生态兼容

```

┌───────────┐    生成配置     ┌────────────────────┐
```

## 3. 关键技术决策

### 3.1 Feature Gate 设计

```
Feature Gate 矩阵：
┌─────────────────────┬──────────┬──────────┬──────────┐
│ Feature             │ 默认状态  │ 成熟度   │ 依赖     │
├─────────────────────┼──────────┼──────────┼──────────┤
│ KubeadmControlPlane │ false    │ Alpha    │ 无       │
│ KubeadmConfigTemplate│ false   │ Alpha    │ KCP      │
│ PreserveBKEFeatures │ true     │ Beta     │ 无       │
└─────────────────────┴──────────┴──────────┴──────────┘

Feature Gate 组合：
┌─────────────────────────────────────────────────────────┐
│ 组合 1：保守模式（默认）                                 │
│   PreserveBKEFeatures=true                              │
│   → 使用旧的 Phase 流程，保留所有 BKE 特性              │
│                                                         │
│ 组合 2：激进模式                                         │
│   KubeadmControlPlane=true                              │
│   KubeadmConfigTemplate=true                            │
│   PreserveBKEFeatures=false                             │
│   → 完全使用 CAPI 标准，牺牲部分 BKE 特性               │
│                                                         │
│ 组合 3：混合模式（推荐）                                 │
│   KubeadmControlPlane=true                              │
│   KubeadmConfigTemplate=true                            │
│   PreserveBKEFeatures=true                              │
│   → 使用 CAPI 标准，同时保留 BKE 特性                   │
└─────────────────────────────────────────────────────────┘
```
**核心思路**：
- **默认保守**：默认不启用新特性，确保稳定性
- **灵活组合**：支持多种 Feature 组合，满足不同场景
- **渐进成熟**：从 Alpha 到 Beta 到 GA，逐步稳定

### 3.3 证书管理策略

```
证书管理演进：

│                                                         │
│  EnsureCerts                                            │
│  ├── 双 CA 模式（自签名 CA + 用户自定义 CA）            │

└─────────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────┐
│ 目标策略：BKE 生成 + CAPI 使用                           │
│                                                         │
│  EnsureCerts（保留）                                    │
│  ├── 双 CA 模式（保留）                                 │
│  ├── 节点感知 SANs（保留）                              │
│  └── 存储为 Secret（而非直接下发）                      │
│                                                         │

│  ├── 读取 BKE 生成的证书 Secret                        │
```

**核心思路**：
- **生成权保留**：BKE 保留证书生成权，确保双 CA 模式和节点感知 SANs
- **使用权移交**：CAPI 使用 BKE 生成的证书，而非自己生成
- **注入机制**：通过 CertInjector 将 BKE 证书注入到 CAPI

## 4. 风险控制策略

### 4.1 分阶段验证

```
阶段验证矩阵：
┌─────────┬──────────────────┬──────────────────┬──────────┐
│ 阶段    │ 验证内容          │ 验证标准          │ 回滚条件 │
├─────────┼──────────────────┼──────────────────┼──────────┤
│ 阶段 1  │ Feature Gate     │ 正常启用/禁用     │ 单元测试 │
│         │ 新 Phase         │ 正常执行          │ 失败     │
├─────────┼──────────────────┼──────────────────┼──────────┤
│ 阶段 2  │ KCP 创建         │ 正确配置          │ E2E 测试 │
│         │ 控制平面初始化    │ 所有节点就绪      │ 失败     │
│         │ 证书注入          │ 正确注入          │ 失败     │
├─────────┼──────────────────┼──────────────────┼──────────┤
│ 阶段 3  │ KCT 创建         │ 正确配置          │ E2E 测试 │
```

**核心思路**：
- **每阶段验证**：每个阶段完成后必须通过验证
- **失败即回滚**：验证失败立即回滚，不进入下一阶段
- **多维度验证**：单元测试、集成测试、E2E 测试、性能测试

- **条件执行**：Phase 根据 Feature Gate 决定执行新逻辑还是旧逻辑

## 5. 核心重构模式

### 5.1 适配器模式

```
适配器模式应用：
┌─────────────────────────────────────────────────────────┐
│                                                         │
│  BKE 特有能力          适配器           CAPI 标准能力  │
│                                                         │
│  ┌──────────┐        ┌──────────┐      ┌──────────┐   │
│  │ 双 CA    │        │ Cert     │      │ Kubeadm  │   │
│  │ 证书     │ ─────→ │ Injector │ ───→ │ Control  │   │
│  └──────────┘        └──────────┘      │ Plane     │   │
│                                        └──────────┘   │
│  ┌──────────┐        ┌──────────┐      ┌──────────┐   │
│  │ 自定义   │        │ Script   │      │ Kubeadm  │   │
│  │ 脚本     │ ─────→ │ Injector │ ───→ │ Config   │   │
│  └──────────┘        └──────────┘      │ Template │   │
│                                        └──────────┘   │
│  ┌──────────┐        ┌──────────┐      ┌──────────┐   │
│  │ 节点环境 │        │ Config   │      │ Machine  │   │
│  │ 配置     │ ─────→ │ Generator│ ───→ │ Template │   │
│  └──────────┘        └──────────┘      └──────────┘   │
│                                                         │
└─────────────────────────────────────────────────────────┘

适配器职责：
┌─────────────────────────────────────────────────────────┐
│ CertInjector：                                          │
│ ├── 读取 BKE 生成的证书 Secret                         │
│ ├── 转换为 CAPI 需要的格式                              │
│ └── 注入到 KubeadmControlPlane                          │
│                                                         │
│ ScriptInjector：                                        │
│ ├── 从 ConfigMap 读取自定义脚本                        │

│ ConfigGenerator：                                       │
│ ├── 读取 BKECluster 配置                               │

└─────────────────────────────────────────────────────────┘
```
**核心思路**：
- **解耦**：通过适配器解耦 BKE 特有能力和 CAPI 标准能力
- **转换**：适配器负责数据格式转换和注入
- **可扩展**：新增适配器即可支持新的能力集成

```
策略模式应用：
┌─────────────────────────────────────────────────────────┐
│                                                         │

│  │   └── NeedExecute()                                  │

│  │                                                      │
│  └── CAPIStrategy（新策略）                             │
│      ├── 使用 CAPI 标准                                 │
│      └── 声明式管理                                    │
│                                                         │
└─────────────────────────────────────────────────────────┘

策略选择：
┌─────────────────────────────────────────────────────────┐
│ func (p *Phase) Execute() error {                       │
│     strategy := p.selectStrategy()                      │
│     return strategy.Execute()                           │
│ }                                                       │
│                                                         │
│ func (p *Phase) selectStrategy() Strategy {             │
│     if feature.IsFeatureEnabled(KubeadmControlPlane) {  │
```

**核心思路**：
- **策略抽象**：定义策略接口，支持多种实现
- **动态选择**：运行时根据 Feature Gate 选择策略
- **易于扩展**：新增策略无需修改现有代码

| 决策点 | 选择 | 理由 |
|--------|------|------|
| **是否保留 BKE 特有能力** | ✅ 保留 | 企业级场景的核心价值 |
| **是否使用 CAPI 标准** | ✅ 使用 | 生态兼容性和维护性 |
| **如何连接两者** | 适配器模式 | 解耦、可扩展 |
| **如何控制切换** | Feature Gate | 灵活、可回滚 |
| **如何保障稳定** | 分阶段验证 | 降低风险 |
| **如何保障兼容** | 新旧共存 | 平滑迁移 |
