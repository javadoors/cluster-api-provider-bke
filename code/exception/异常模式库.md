## 一、模式库概述

### 1.1 目的

异常模式库提供可复用的异常处理模式，用于：
1. 统一异常处理风格
2. 提高代码可读性和可维护性
3. 减少重复代码
4. 确保异常处理完整性
5. 加速开发过程

### 1.2 模式分类

| 分类 | 说明 | 模式数量 |
|------|------|---------|
| **基础模式** | 单一异常场景处理 | 8个 |
| **组合模式** | 多异常场景组合处理 | 4个 |
| **高级模式** | 复杂异常处理流程 | 3个 |

### 1.3 使用指南

**[MUST]** 使用模式库时必须：
1. 选择正确的模式类型
2. 遵循模式实现规范
3. 保持模式完整性
4. 添加必要的上下文信息

**[SHOULD]** 模式使用建议：
1. 优先使用基础模式
2. 复杂场景使用组合模式
3. 特殊场景使用高级模式
4. 避免过度设计

## 二、基础模式

### 模式 1: 资源不存在模式

**模式标识**: `PATTERN-NOT-FOUND`  
**适用场景**: K8s资源、配置文件、证书等不存在  
**严重程度**: Info 或 Error（视场景而定）

#### 识别特征

```go
// 特征1: apierrors.IsNotFound
if apierrors.IsNotFound(err) {
    // 资源不存在
}

// 特征2: 错误信息包含 "not found"
if strings.Contains(err.Error(), "not found") {
    // 资源不存在
}
```

#### 标准实现

```go
// 模式实现: 资源不存在处理
func (e *Phase) getResource() (Resource, error) {
    resource, err := e.Client.Get(e.Ctx, key)
    if err != nil {
        // [PATTERN-NOT-FOUND]
        if apierrors.IsNotFound(err) {
            // 场景1: 资源不存在是正常情况（如首次创建）
            log.Info("Resource not found, will create", "key", key)
            return nil, nil  // 返回nil，触发创建逻辑
            
            // 或场景2: 资源不存在是异常情况（如依赖资源缺失）
            // return nil, errors.Errorf("required resource %s not found", key)
        }
        
        // 其他错误
        return nil, errors.Wrap(err, "failed to get resource")
    }
    
    return resource, nil
}
```

#### 处理策略矩阵

| 场景 | 资源类型 | 处理方式 | 返回值 |
|------|---------|---------|--------|
| 首次创建 | 目标资源 | 正常，触发创建 | `nil, nil` |
| 依赖缺失 | 依赖资源 | 异常，需等待 | `nil, err` + Requeue |
| 已删除 | 已删除资源 | 正常，结束流程 | `nil, nil` |
| 意外丢失 | 关键资源 | 异常，需重建 | `nil, err` + 标记重建 |

#### 使用示例

```go
// 示例1: 获取Command（不存在则创建）
command, err := phaseutil.GetMasterInitCommand(ctx, client, cluster)
if err != nil {
    if strings.Contains(err.Error(), "command not found") {
        // [PATTERN-NOT-FOUND] Command不存在，需要创建
        log.Info("Init command not found, will create")
        return e.createInitCommand()
    }
    return nil, err
}

// 示例2: 获取Secret（不存在则失败）
secret, err := e.Client.Get(ctx, secretKey)
if err != nil {
    if apierrors.IsNotFound(err) {
        // [PATTERN-NOT-FOUND] Secret不存在，无法继续
        return ctrl.Result{}, errors.Errorf("certificate secret %s not found", secretKey)
    }
    return ctrl.Result{}, err
}
```

### 模式 2: 网络连接失败模式

**模式标识**: `PATTERN-NETWORK-ERROR`  
**适用场景**: SSH连接、API调用、网络超时  
**严重程度**: Error（可恢复）

#### 识别特征

```go
// 特征1: 连接拒绝
if strings.Contains(err.Error(), "connection refused") {
    // 网络不可达
}

// 特征2: 超时
if errors.Is(err, context.DeadlineExceeded) {
    // 网络超时
}

// 特征3: SSH特定错误
if strings.Contains(err.Error(), "ssh:") {
    // SSH连接失败
}
```

#### 标准实现

```go
// 模式实现: 网络错误处理
func (e *Phase) connectNode(nodeIP string) error {
    sshClient, err := ssh.NewSSHClient(nodeIP)
    if err != nil {
        // [PATTERN-NETWORK-ERROR]
        
        // 判断错误类型
        if isTemporaryNetworkError(err) {
            // 临时性网络错误，Requeue重试
            log.Warn("Network connection failed, will retry",
                "node", nodeIP,
                "error", err)
            return err  // 触发Requeue
        }
        
        // 永久性网络错误（如认证失败）
        log.Error("Network connection permanently failed",
            "node", nodeIP,
            "error", err)
        return err
    }
    
    return nil
}

// 判断是否为临时性网络错误
func isTemporaryNetworkError(err error) bool {
    errStr := err.Error()
    return strings.Contains(errStr, "connection refused") ||
           strings.Contains(errStr, "timeout") ||
           strings.Contains(errStr, "temporary failure")
}
```

#### 处理策略

| 错误类型 | 判断条件 | 处理方式 | Requeue间隔 |
|---------|---------|---------|------------|
| 连接拒绝 | "connection refused" | Requeue重试 | 10秒 |
| 超时 | DeadlineExceeded | Requeue重试 | 10秒 |
| 认证失败 | "authentication failed" | 标记失败 | 不Requeue |
| 主机不可达 | "no route to host" | Requeue重试 | 30秒 |

### 模式 3: 命令执行失败模式

**模式标识**: `PATTERN-COMMAND-FAILED`  
**适用场景**: Shell命令、Kubeadm命令、脚本执行失败  
**严重程度**: Error

#### 识别特征

```go
// 特征1: Command Phase = Failed
if command.Status.Phase == agentv1beta1.CommandFailed {
    // 命令执行失败
}

// 特征2: Condition Status = False
for _, condition := range command.Status.Conditions {
    if condition.Status == metav1.ConditionFalse {
        // 该命令步骤失败
    }
}

// 特征3: 命令返回码非0
if exitCode != 0 {
    // 命令执行失败
}
```

#### 标准实现

```go
// 模式实现: 命令执行失败处理
func (e *Phase) executeCommand(node Node) error {
    // 创建命令
    cmd := &command.BaseCommand{
        Ctx:             ctx,
        WaitTimeout:     10 * time.Minute,
        BackoffLimit:    3,  // 最多重试3次
    }
    
    if err := cmd.Create(); err != nil {
        return errors.Wrap(err, "failed to create command")
    }
    
    // 等待命令完成
    complete, successNodes, failedNodes := cmd.Wait()
    
    // [PATTERN-COMMAND-FAILED]
    if !complete {
        // 超时
        log.Error("Command execution timeout",
            "node", node.IP,
            "timeout", cmd.WaitTimeout)
        return errors.Errorf("command timeout after %v", cmd.WaitTimeout)
    }
    
    if len(failedNodes) > 0 {
        // 执行失败
        log.Error("Command execution failed",
            "node", node.IP,
            "failedNodes", failedNodes)
        return errors.Errorf("command failed on nodes: %v", failedNodes)
    }
    
    return nil
}
```

#### 重试策略

```go
// 命令重试配置
type CommandRetryConfig struct {
    BackoffLimit    int   // 最大重试次数
    BackoffDelay    int   // 重试间隔（秒）
    BackoffIgnore   bool  // 失败是否跳过
}

// 根据命令类型选择重试策略
func getRetryConfig(cmdType string) CommandRetryConfig {
    switch cmdType {
    case "kubeadm-init":
        // [PATTERN-COMMAND-FAILED] Master初始化不重试
        return CommandRetryConfig{
            BackoffLimit:  0,
            BackoffIgnore: false,
        }
        
    case "kubeadm-join":
        // [PATTERN-COMMAND-FAILED] 节点加入重试3次
        return CommandRetryConfig{
            BackoffLimit:  3,
            BackoffDelay:  5,
            BackoffIgnore: false,
        }
        
    case "env-init":
        // [PATTERN-COMMAND-FAILED] 环境初始化重试3次
        return CommandRetryConfig{
            BackoffLimit:  3,
            BackoffDelay:  10,
            BackoffIgnore: false,
        }
        
    case "non-critical":
        // [PATTERN-COMMAND-FAILED] 非关键命令可跳过
        return CommandRetryConfig{
            BackoffLimit:  1,
            BackoffDelay:  2,
            BackoffIgnore: true,  // 失败后跳过
        }
        
    default:
        return CommandRetryConfig{
            BackoffLimit:  3,
            BackoffDelay:  5,
            BackoffIgnore: false,
        }
    }
}
```

### 模式 4: 状态持久化失败模式

**模式标识**: `PATTERN-STATUS-PERSIST`  
**适用场景**: 状态更新、Condition设置、Status同步  
**严重程度**: Error

#### 识别特征

```go
// 特征: SyncStatusUntilComplete 返回错误
if err := mergecluster.SyncStatusUntilComplete(client, bkeCluster); err != nil {
    // 状态持久化失败
}
```

#### 标准实现

```go
// 模式实现: 状态持久化
func (e *Phase) updateStatus() error {
    // 1. 修改状态
    e.BKECluster.Status.Phase = newPhase
    e.BKECluster.Status.Reason = reason
    
    // 2. [PATTERN-STATUS-PERSIST] 持久化状态
    if err := mergecluster.SyncStatusUntilComplete(e.Client, e.BKECluster); err != nil {
        // 状态持久化失败，必须返回错误触发Requeue
        log.Error("Failed to persist status",
            "phase", newPhase,
            "error", err)
        return errors.Wrap(err, "failed to sync status")
    }
    
    // 3. 刷新上下文
    if err := e.Ctx.RefreshCtxBKECluster(); err != nil {
        return errors.Wrap(err, "failed to refresh context")
    }
    
    return nil
}

// 带重试的状态持久化
func (e *Phase) updateStatusWithRetry() error {
    maxRetries := 3
    for i := 0; i < maxRetries; i++ {
        // [PATTERN-STATUS-PERSIST] 尝试持久化
        if err := mergecluster.SyncStatusUntilComplete(e.Client, e.BKECluster); err != nil {
            if i == maxRetries-1 {
                // 最后一次重试仍失败
                return errors.Wrap(err, "failed to sync status after retries")
            }
            
            log.Warn("Status sync failed, will retry",
                "attempt", i+1,
                "error", err)
            time.Sleep(time.Duration(i+1) * time.Second)  // 指数退避
            continue
        }
        
        return nil  // 成功
    }
    
    return errors.New("unreachable")
}
```

#### 处理原则

**[MUST]** 状态持久化失败必须：
1. 立即返回错误
2. 触发Requeue
3. 不继续执行后续操作
4. 记录Error级别日志

### 模式 5: 超时模式

**模式标识**: `PATTERN-TIMEOUT`  
**适用场景**: 操作超时、轮询超时、等待超时  
**严重程度**: Error

#### 识别特征

```go
// 特征1: context.DeadlineExceeded
if errors.Is(err, context.DeadlineExceeded) {
    // 超时
}

// 特征2: wait.ErrWaitTimeout
if errors.Is(err, wait.ErrWaitTimeout) {
    // 轮询超时
}
```

#### 标准实现

```go
// 模式实现: 轮询超时处理
func (e *Phase) waitForCondition() error {
    timeout := 5 * time.Minute
    pollInterval := 2 * time.Second
    
    // [PATTERN-TIMEOUT] 设置超时上下文
    ctx, cancel := context.WithTimeout(e.Ctx, timeout)
    defer cancel()
    
    err := wait.PollUntilContextTimeout(
        ctx,
        pollInterval,
        timeout,
        true,
        func(ctx context.Context) (bool, error) {
            // 检查条件
            satisfied, err := e.checkCondition()
            if err != nil {
                return false, nil  // 继续轮询
            }
            return satisfied, nil
        },
    )
    
    // [PATTERN-TIMEOUT] 处理超时
    if err != nil {
        if errors.Is(err, context.DeadlineExceeded) {
            log.Error("Operation timeout",
                "timeout", timeout,
                "operation", "waitForCondition")
            return errors.Errorf("timeout after %v waiting for condition", timeout)
        }
        return err
    }
    
    return nil
}

// 命令执行超时处理
func (e *Phase) handleCommandTimeout(cmd *agentv1beta1.Command) error {
    // [PATTERN-TIMEOUT] 检查是否超时
    if cmd.Status.LastStartTime != nil {
        elapsed := time.Since(cmd.Status.LastStartTime.Time)
        deadline := time.Duration(cmd.Spec.ActiveDeadlineSecond) * time.Second
        
        if elapsed > deadline {
            log.Error("Command execution timeout",
                "command", cmd.Name,
                "elapsed", elapsed,
                "deadline", deadline)
            return errors.Errorf("command timeout after %v", elapsed)
        }
    }
    
    return nil
}
```

#### 超时配置表

| 操作类型 | 推荐超时 | 计算依据 |
|---------|---------|---------|
| API调用 | 30秒 | 网络延迟+API处理 |
| SSH命令 | 60秒 | 命令执行+网络传输 |
| 服务启动 | 5分钟 | 服务启动+健康检查 |
| 节点加入 | 10分钟 | Kubeadm+证书+组件启动 |
| 集群初始化 | 15分钟 | 多组件启动+健康检查 |

### 模式 6: 版本冲突模式

**模式标识**: `PATTERN-CONFLICT`  
**适用场景**: 资源更新冲突、乐观锁冲突  
**严重程度**: Warning（可自动恢复）

#### 识别特征

```go
// 特征: apierrors.IsConflict
if apierrors.IsConflict(err) {
    // 版本冲突
}
```

#### 标准实现

```go
// 模式实现: 版本冲突处理
func (e *Phase) updateResource(resource *Resource) error {
    // [PATTERN-CONFLICT] 使用PatchHelper自动处理冲突
    helper, err := patch.NewHelper(resource, e.Client)
    if err != nil {
        return errors.Wrap(err, "failed to create patch helper")
    }
    
    // 修改资源
    resource.Spec.Field = newValue
    
    // Patch会自动重试冲突
    if err := helper.Patch(e.Ctx, resource); err != nil {
        if apierrors.IsConflict(err) {
            // [PATTERN-CONFLICT] 仍冲突，Requeue
            log.Warn("Resource update conflict, will retry",
                "resource", resource.Name)
            return err  // 触发Requeue
        }
        return errors.Wrap(err, "failed to patch resource")
    }
    
    return nil
}

// 手动重试冲突
func (e *Phase) updateWithRetry(resource *Resource) error {
    maxRetries := 3
    for i := 0; i < maxRetries; i++ {
        // 获取最新版本
        current, err := e.getLatest(resource.Name)
        if err != nil {
            return err
        }
        
        // 应用修改
        current.Spec.Field = newValue
        
        // 尝试更新
        if err := e.Client.Update(e.Ctx, current); err != nil {
            if apierrors.IsConflict(err) {
                // [PATTERN-CONFLICT] 版本冲突，重试
                if i < maxRetries-1 {
                    log.Warn("Update conflict, will retry",
                        "attempt", i+1)
                    time.Sleep(time.Second)
                    continue
                }
            }
            return err
        }
        
        return nil
    }
    
    return errors.New("max retries exceeded")
}
```

### 模式 7: 前置条件不满足模式

**模式标识**: `PATTERN-PRECONDITION`  
**适用场景**: 依赖资源未就绪、条件检查失败  
**严重程度**: Warning

#### 识别特征

```go
// 特征: 条件检查返回false
if !conditions.IsTrue(cluster, clusterv1.ControlPlaneInitializedCondition) {
    // 前置条件不满足
}

// 特征: 状态标志未设置
if node.Status.StateCode & bkev1beta1.NodeAgentReadyFlag == 0 {
    // Agent未就绪
}
```

#### 标准实现

```go
// 模式实现: 前置条件检查
func (e *Phase) Execute() (ctrl.Result, error) {
    // [PATTERN-PRECONDITION] 检查前置条件
    if !e.checkPreconditions() {
        log.Info("Preconditions not met, waiting",
            "phase", e.Name())
        return ctrl.Result{RequeueAfter: 10 * time.Second}, nil
    }
    
    // 前置条件满足，继续执行
    return e.doExecute()
}

// 前置条件检查函数
func (e *Phase) checkPreconditions() bool {
    // 1. 检查集群状态
    if !conditions.IsTrue(e.Cluster, clusterv1.ControlPlaneInitializedCondition) {
        log.Info("Control plane not initialized")
        return false
    }
    
    // 2. 检查节点状态
    nodes := e.getNodes()
    for _, node := range nodes {
        if node.Status.StateCode & bkev1beta1.NodeAgentReadyFlag == 0 {
            log.Info("Agent not ready", "node", node.IP)
            return false
        }
    }
    
    // 3. 检查依赖资源
    if !e.checkDependencies() {
        return false
    }
    
    return true
}

// 多条件组合检查
func (e *Phase) checkMultipleConditions() (bool, ctrl.Result, error) {
    conditions := []struct {
        name string
        check func() bool
    }{
        {"ClusterInitialized", e.isClusterInitialized},
        {"AgentsReady", e.areAgentsReady},
        {"CertificatesReady", e.areCertificatesReady},
        {"NetworkAvailable", e.isNetworkAvailable},
    }
    
    for _, cond := range conditions {
        if !cond.check() {
            // [PATTERN-PRECONDITION] 条件不满足
            log.Info("Precondition not satisfied",
                "condition", cond.name)
            return false, ctrl.Result{RequeueAfter: 10 * time.Second}, nil
        }
    }
    
    return true, ctrl.Result{}, nil
}
```

### 模式 8: 配置错误模式

**模式标识**: `PATTERN-CONFIG-ERROR`  
**适用场景**: 配置缺失、配置无效、版本不支持  
**严重程度**: Error（不可恢复）

#### 识别特征

```go
// 特征1: 配置为空
if config == nil {
    // 配置缺失
}

// 特征2: 配置校验失败
if err := validate(config); err != nil {
    // 配置无效
}

// 特征3: 版本不支持
if !isSupportedVersion(version) {
    // 版本不支持
}
```

#### 标准实现

```go
// 模式实现: 配置校验
func (e *Phase) validateConfig(config *Config) error {
    // [PATTERN-CONFIG-ERROR] 配置存在性检查
    if config == nil {
        return errors.New("config is nil")
    }
    
    // [PATTERN-CONFIG-ERROR] 必填字段检查
    if config.ClusterName == "" {
        return errors.New("cluster name is required")
    }
    if config.KubernetesVersion == "" {
        return errors.New("kubernetes version is required")
    }
    
    // [PATTERN-CONFIG-ERROR] 版本支持检查
    if !e.isSupportedVersion(config.KubernetesVersion) {
        return errors.Errorf("unsupported kubernetes version: %s", 
            config.KubernetesVersion)
    }
    
    // [PATTERN-CONFIG-ERROR] 数值范围检查
    if config.NodeCount < 1 || config.NodeCount > 1000 {
        return errors.Errorf("invalid node count: %d, must be 1-1000", 
            config.NodeCount)
    }
    
    // [PATTERN-CONFIG-ERROR] 依赖关系检查
    if config.ExternalEtcd != nil && config.LocalEtcd != nil {
        return errors.New("cannot specify both external and local etcd")
    }
    
    return nil
}

// 配置错误处理
func (e *Phase) Execute() (ctrl.Result, error) {
    config := e.BKECluster.Spec.ClusterConfig
    
    // [PATTERN-CONFIG-ERROR] 校验配置
    if err := e.validateConfig(config); err != nil {
        // 配置错误，无法自动恢复
        log.Error("Configuration validation failed",
            "error", err,
            "cluster", e.BKECluster.Name)
        
        // 标记集群状态为Failed
        condition.ConditionMark(e.BKECluster, 
            bkev1beta1.ClusterValidationFailedCondition,
            confv1beta1.ConditionFalse,
            constant.ConfigurationInvalidReason,
            err.Error())
        
        // 返回错误，不Requeue（配置错误需人工修正）
        return ctrl.Result{}, err
    }
    
    // 配置有效，继续执行
    return e.doExecute()
}
```

## 三、组合模式

### 模式 9: Fail-Fast模式

**模式标识**: `PATTERN-FAIL-FAST`  
**适用场景**: Master初始化、Etcd操作、控制平面关键操作  
**严重程度**: Fatal

#### 标准实现

```go
// 模式实现: Fail-Fast策略
func (e *EnsureMasterInit) Execute() (ctrl.Result, error) {
    ctx, c, bkeCluster, _, log := e.Ctx.Untie()
    
    // [PATTERN-FAIL-FAST] 前置检查
    nodes, err := e.validateMasterNodes()
    if err != nil {
        // 无Master节点，立即失败
        log.Error(constant.MasterNotInitReason, 
            "No master nodes available: %v", err)
        return ctrl.Result{}, err  // 不Requeue
    }
    
    // [PATTERN-FAIL-FAST] 执行初始化
    if err := e.initializeMaster(nodes[0]); err != nil {
        // 标记失败状态
        e.markNodeFailed(nodes[0], err)
        if syncErr := mergecluster.SyncStatusUntilComplete(c, bkeCluster); syncErr != nil {
            return ctrl.Result{}, syncErr
        }
        
        // 记录Fatal级别日志
        log.Error(constant.MasterInitFailedReason,
            "Master initialization failed, requires manual intervention",
            "node", nodes[0].IP,
            "error", err)
        
        // [PATTERN-FAIL-FAST] 立即返回，不重试
        return ctrl.Result{}, err
    }
    
    return ctrl.Result{}, nil
}

// Fail-Fast命令配置
func (e *Phase) createFailFastCommand(node Node) error {
    cmd := &command.BaseCommand{
        Ctx:          ctx,
        WaitTimeout:  15 * time.Minute,
        BackoffLimit: 0,  // [PATTERN-FAIL-FAST] 不重试
    }
    
    // ... 创建命令
}
```

#### 关键特征

**[MUST]** Fail-Fast模式必须：
1. BackoffLimit = 0（不重试）
2. 立即返回错误（不Requeue）
3. 记录Fatal级别日志
4. 标记节点/集群状态为Failed
5. 提供人工介入指引

### 模式 10: Best-Effort模式

**模式标识**: `PATTERN-BEST-EFFORT`  
**适用场景**: Worker节点操作、非关键组件部署  
**严重程度**: Error（部分失败）

#### 标准实现

```go
// 模式实现: Best-Effort策略
func (e *EnsureWorkerJoin) Execute() (ctrl.Result, error) {
    ctx, c, bkeCluster, _, log := e.Ctx.Untie()
    
    nodes := e.getWorkerNodes()
    var failedNodes []string
    
    for _, node := range nodes {
        // [PATTERN-BEST-EFFORT] 标记开始
        e.markNodeJoining(node)
        
        // 执行加入
        err := e.joinWorker(node)
        
        // [PATTERN-BEST-EFFORT] 处理结果
        if err != nil {
            // 记录失败节点
            failedNodes = append(failedNodes, node.IP)
            
            // 标记单个节点失败
            e.markNodeFailed(node, err)
            
            // 记录Warning日志
            log.Warn(constant.WorkerJoinFailedReason,
                "Worker join failed, continuing with other nodes",
                "node", node.IP,
                "error", err)
            
            // [PATTERN-BEST-EFFORT] 继续处理下一个节点
            continue
        }
        
        // 标记成功
        e.markNodeReady(node)
    }
    
    // [PATTERN-BEST-EFFORT] 持久化所有状态
    if err := mergecluster.SyncStatusUntilComplete(c, bkeCluster); err != nil {
        return ctrl.Result{}, err
    }
    
    // [PATTERN-BEST-EFFORT] 如有失败节点，返回错误触发Requeue
    if len(failedNodes) > 0 {
        log.Warn("Some workers failed to join",
            "failedNodes", failedNodes,
            "totalNodes", len(nodes))
        return ctrl.Result{}, errors.Errorf(
            "%d/%d workers failed: %v", 
            len(failedNodes), len(nodes), failedNodes)
    }
    
    return ctrl.Result{}, nil
}
```

#### 关键特征

**[MUST]** Best-Effort模式必须：
1. 记录所有失败资源
2. 继续处理其他资源
3. 最后统一持久化状态
4. 如有失败，返回错误触发Requeue
5. 记录失败统计信息

### 模式 11: 轮询等待模式

**模式标识**: `PATTERN-POLL-WAIT`  
**适用场景**: 等待条件满足、健康检查、状态同步  
**严重程度**: Info → Error（超时时）

#### 标准实现

```go
// 模式实现: 轮询等待
func (e *Phase) waitForReady() error {
    timeout := 5 * time.Minute
    pollInterval := 2 * time.Second
    logInterval := 10  // 每10次轮询输出一次日志
    
    // [PATTERN-POLL-WAIT] 轮询等待
    err := wait.PollUntilContextTimeout(
        e.Ctx,
        pollInterval,
        timeout,
        true,
        func(ctx context.Context) (bool, error) {
            pollCount++
            
            // 检查条件
            ready, err := e.checkReady()
            if err != nil {
                // [PATTERN-POLL-WAIT] 错误时控制日志频率
                if pollCount % logInterval == 0 {
                    log.Warn("Health check error",
                        "error", err,
                        "pollCount", pollCount)
                }
                return false, nil  // 继续轮询
            }
            
            if ready {
                // 条件满足
                log.Info("Condition satisfied",
                    "pollCount", pollCount)
                return true, nil
            }
            
            // [PATTERN-POLL-WAIT] 未满足时控制日志频率
            if pollCount % logInterval == 0 {
                log.Info("Condition not satisfied, waiting",
                    "pollCount", pollCount)
            }
            return false, nil
        },
    )
    
    // [PATTERN-POLL-WAIT] 处理超时
    if err != nil {
        if errors.Is(err, context.DeadlineExceeded) {
            return errors.Errorf("timeout after %v", timeout)
        }
        return err
    }
    
    return nil
}

// 多条件轮询等待
func (e *Phase) waitForMultipleConditions() error {
    conditions := []struct {
        name string
        check func() (bool, error)
    }{
        {"APIServerReady", e.isAPIServerReady},
        {"EtcdHealthy", e.isEtcdHealthy},
        {"NodesReady", e.areNodesReady},
    }
    
    for _, cond := range conditions {
        // [PATTERN-POLL-WAIT] 等待每个条件
        if err := e.waitForCondition(cond.check); err != nil {
            return errors.Wrapf(err, "waiting for %s", cond.name)
        }
    }
    
    return nil
}
```

#### 关键特征

**[MUST]** 轮询等待模式必须：
1. 设置合理的超时时间
2. 设置合理的轮询间隔
3. 控制日志输出频率
4. 正确处理超时错误
5. 错误时继续轮询（除非致命错误）

### 模式 12: 重试耗尽模式

**模式标识**: `PATTERN-RETRY-EXHAUSTED`  
**适用场景**: 达到最大重试次数、无法恢复  
**严重程度**: Error

#### 标准实现

```go
// 模式实现: 重试耗尽处理
func (e *Phase) handleRetryExhausted(cmd *agentv1beta1.Command, node Node) error {
    // [PATTERN-RETRY-EXHAUSTED] 检查重试次数
    retryCount := e.getRetryCount(cmd)
    
    if retryCount >= cmd.Spec.BackoffLimit {
        // 达到最大重试次数
        log.Error("Retry limit exhausted",
            "node", node.IP,
            "command", cmd.Name,
            "retryCount", retryCount,
            "limit", cmd.Spec.BackoffLimit)
        
        // [PATTERN-RETRY-EXHAUSTED] 检查是否可跳过
        for _, execCmd := range cmd.Spec.Commands {
            if execCmd.BackoffIgnore {
                // 可跳过，标记为Skip
                log.Warn("Command failed but marked as ignorable, skipping",
                    "command", execCmd.ID)
                e.markCommandSkipped(cmd, execCmd.ID)
                continue
            }
            
            // 不可跳过，标记为Failed
            e.markCommandFailed(cmd, execCmd.ID)
        }
        
        // [PATTERN-RETRY-EXHAUSTED] 判断是否需人工介入
        if e.requiresManualIntervention(cmd) {
            log.Error("Manual intervention required",
                "node", node.IP,
                "reason", "retry exhausted")
            return errors.Errorf(
                "operation failed after %d retries on node %s, requires manual intervention",
                retryCount, node.IP)
        }
        
        // 可自动恢复（如重新创建命令）
        return e.recoverFromFailure(cmd, node)
    }
    
    // 未达到最大重试次数，继续重试
    return e.retryCommand(cmd, node)
}
```

### 模式 13: 分层异常处理模式

**模式标识**: `PATTERN-LAYERED-HANDLING`  
**适用场景**: 复杂操作流程、多阶段任务  
**严重程度**: 混合

#### 标准实现

```go
// 模式实现: 分层异常处理
func (e *Phase) Execute() (ctrl.Result, error) {
    // [PATTERN-LAYERED-HANDLING] Layer 1: 前置检查层
    if result, err := e.preCheckLayer(); err != nil {
        return result, err
    }
    
    // [PATTERN-LAYERED-HANDLING] Layer 2: 资源准备层
    resources, result, err := e.prepareLayer()
    if err != nil {
        return result, err
    }
    
    // [PATTERN-LAYERED-HANDLING] Layer 3: 核心执行层
    if result, err := e.executeLayer(resources); err != nil {
        return result, err
    }
    
    // [PATTERN-LAYERED-HANDLING] Layer 4: 后置处理层
    if err := e.postProcessLayer(); err != nil {
        // 后置处理失败不影响主流程
        log.Warn("Post process failed", "error", err)
    }
    
    return ctrl.Result{}, nil
}

// Layer 1: 前置检查层
func (e *Phase) preCheckLayer() (ctrl.Result, error) {
    // 检查集群状态
    if !e.isClusterReady() {
        return ctrl.Result{RequeueAfter: 10 * time.Second}, nil
    }
    
    // 检查节点状态
    if !e.areNodesReady() {
        return ctrl.Result{RequeueAfter: 10 * time.Second}, nil
    }
    
    return ctrl.Result{}, nil
}

// Layer 2: 资源准备层
func (e *Phase) prepareLayer() (Resources, ctrl.Result, error) {
    // 获取配置
    config, err := e.getConfig()
    if err != nil {
        // [PATTERN-CONFIG-ERROR]
        return nil, ctrl.Result{}, err
    }
    
    // 获取证书
    certs, err := e.getCertificates()
    if err != nil {
        // [PATTERN-NOT-FOUND]
        return nil, ctrl.Result{RequeueAfter: 10 * time.Second}, nil
    }
    
    // 获取节点列表
    nodes, err := e.getNodes()
    if err != nil {
        return nil, ctrl.Result{}, err
    }
    
    return Resources{Config: config, Certs: certs, Nodes: nodes}, 
           ctrl.Result{}, nil
}

// Layer 3: 核心执行层
func (e *Phase) executeLayer(resources Resources) (ctrl.Result, error) {
    for _, node := range resources.Nodes {
        // [PATTERN-COMMAND-FAILED]
        if err := e.executeOnNode(node, resources); err != nil {
            // 根据策略处理
            if e.shouldFailFast() {
                // [PATTERN-FAIL-FAST]
                return ctrl.Result{}, err
            }
            // [PATTERN-BEST-EFFORT]
            e.recordFailure(node, err)
            continue
        }
    }
    
    return ctrl.Result{}, nil
}

// Layer 4: 后置处理层
func (e *Phase) postProcessLayer() error {
    // 更新状态
    if err := e.updateStatus(); err != nil {
        // [PATTERN-STATUS-PERSIST]
        return err
    }
    
    // 清理资源
    if err := e.cleanup(); err != nil {
        log.Warn("Cleanup failed", "error", err)
    }
    
    return nil
}
```

### 模式 14: 熔断器模式

**模式标识**: `PATTERN-CIRCUIT-BREAKER`  
**适用场景**: 防止级联失败、保护系统稳定性  
**严重程度**: Warning → Error

#### 标准实现

```go
// 熔断器状态
type CircuitBreaker struct {
    failureCount    int
    failureThreshold int
    timeout         time.Duration
    lastFailureTime time.Time
    state           string  // "closed", "open", "half-open"
}

// 模式实现: 熔断器
func (e *Phase) executeWithCircuitBreaker(node Node) error {
    cb := e.getCircuitBreaker(node.IP)
    
    // [PATTERN-CIRCUIT-BREAKER] 检查熔断器状态
    if cb.state == "open" {
        // 熔断器打开，快速失败
        if time.Since(cb.lastFailureTime) < cb.timeout {
            log.Warn("Circuit breaker is open, skipping operation",
                "node", node.IP,
                "timeSinceFailure", time.Since(cb.lastFailureTime))
            return errors.New("circuit breaker is open")
        }
        
        // 超时后进入半开状态
        cb.state = "half-open"
    }
    
    // 执行操作
    err := e.doOperation(node)
    
    // [PATTERN-CIRCUIT-BREAKER] 更新熔断器状态
    if err != nil {
        cb.failureCount++
        cb.lastFailureTime = time.Now()
        
        if cb.failureCount >= cb.failureThreshold {
            // 达到阈值，打开熔断器
            cb.state = "open"
            log.Error("Circuit breaker opened",
                "node", node.IP,
                "failureCount", cb.failureCount)
        }
        
        return err
    }
    
    // 成功，重置熔断器
    cb.failureCount = 0
    cb.state = "closed"
    
    return nil
}
```

### 模式 15: 降级处理模式

**模式标识**: `PATTERN-DEGRADATION`  
**适用场景**: 非关键功能失败、功能降级  
**严重程度**: Warning

#### 标准实现

```go
// 模式实现: 降级处理
func (e *Phase) executeWithDegradation(node Node) error {
    // 尝试完整功能
    err := e.executeFullFunctionality(node)
    
    if err != nil {
        log.Warn("Full functionality failed, trying degraded mode",
            "node", node.IP,
            "error", err)
        
        // [PATTERN-DEGRADATION] 降级到基本功能
        err = e.executeDegradedFunctionality(node)
        
        if err != nil {
            // 降级也失败
            log.Error("Degraded functionality also failed",
                "node", node.IP,
                "error", err)
            return err
        }
        
        // [PATTERN-DEGRADATION] 标记降级状态
        e.markDegraded(node)
        log.Warn("Operating in degraded mode",
            "node", node.IP)
    }
    
    return nil
}

// 示例: Agent推送降级
func (e *EnsureBKEAgent) pushAgentWithDegradation(node Node) error {
    // 尝试使用RBAC
    err := e.pushAgentWithRBAC(node)
    
    if err != nil {
        // [PATTERN-DEGRADATION] 降级到local kubeconfig
        log.Warn("RBAC push failed, falling back to local kubeconfig",
            "node", node.IP,
            "error", err)
        
        err = e.pushAgentWithLocalKubeconfig(node)
        if err != nil {
            return err
        }
        
        // 标记使用降级模式
        e.markFallbackMode(node)
    }
    
    return nil
}
```
