# 状态机流程完整规格

## 1. 状态机概述

BKE Cluster Provider实现了一套完整的状态机机制，用于管理集群和节点的生命周期状态转换。状态机包含以下核心组件：

1. **集群状态** - 集群整体生命周期状态
2. **Phase状态** - Phase执行状态
3. **节点状态** - 单个节点的生命周期状态
4. **集群健康状态** - 集群健康检查状态
5. **状态管理器** - 状态记录和失败重试机制
6. **Phase流程状态机** - Phase执行流程控制

#### 6.3.1 Phase计算流程

```go
func (p *PhaseFlow) CalculatePhase(old, new *BKECluster) error {
    // Step 1: 确定Phase函数列表
    phasesFuncs := p.determinePhasesFuncs()
    
    // Step 2: 计算并添加需要执行的Phase
    p.calculateAndAddPhases(old, new, phasesFuncs)
    
    // Step 3: 上报Phase状态
    return p.ReportPhaseStatus()
}
```

#### 6.3.2 Phase确定逻辑

```go
func (p *PhaseFlow) determinePhasesFuncs() []PhaseFunc {
    if IsDeleteOrReset(BKECluster) {
        return DeletePhases  // 删除场景
    }
    return FullPhasesRegisFunc  // 正常场景
}
```

#### 6.3.3 Phase执行流程

```go
func (p *PhaseFlow) Execute() (ctrl.Result, error) {
    // Step 1: 确定需要执行的Phase列表
    phases := p.determinePhases()
    
    // Step 2: 启动状态监控
    go p.ctx.WatchBKEClusterStatus()
    
    // Step 3: 执行Phase列表
    return p.executePhases(phases)
}
```

#### 6.3.4 Phase执行详细流程

```go
func (p *PhaseFlow) executePhases(phases) (ctrl.Result, error) {
    for _, phase := range p.BKEPhases {
        if phase.Name().In(phases) {
            // 移除该Phase从待执行列表
            phases.Remove(phase.Name())
            
            // 注册前置Hook
            phase.RegisterPreHooks(
                calculatingClusterPreStatusByPhase,  // 计算前置集群状态
                registerPhaseCName,                  // 注册Phase名称
            )
            
            // 注册后置Hook
            phase.RegisterPostHooks(
                calculatingClusterPostStatusByPhase,  // 计算后置集群状态
            )
            
            // 实时计算是否需要执行
            if phase.NeedExecute(old, new) {
                // 执行前置Hook
                phase.ExecutePreHook()
                
                // 执行Phase
                result, err := phase.Execute()
                
                // 执行后置Hook
                phase.ExecutePostHook(err)
            } else {
                phase.SetStatus(PhaseSkipped)
            }
        } else {
            phase.SetStatus(PhaseSkipped)
        }
    }
}
```

```go
func calculateClusterStatusByPhase(phase Phase, err error) error {
    phaseName := phase.Name()
    ctx := phase.GetPhaseContext()
    
    switch {
        case phaseName.In(CustomSetStatusPhaseNames):
            return nil  // 自定义状态，不处理
            
        case phaseName.In(ClusterInitPhaseNames):
            handleClusterInitPhase(ctx, err)
            
        case phaseName.In(ClusterScaleMasterUpPhaseNames):
            handleClusterScaleMasterUpPhase(ctx, err)
            
        case phaseName.In(ClusterScaleWorkerUpPhaseNames):
            handleClusterScaleWorkerUpPhase(ctx, err)
            
        case phaseName.In(ClusterDeletePhaseNames):
            handleClusterDeletePhase(ctx, err)
            
        case phaseName.In(ClusterPausedPhaseNames):
            handleClusterPausedPhase(ctx, err)
            
        case phaseName.In(ClusterDryRunPhaseNames):
            handleClusterDryRunPhase(ctx, err)
            
        case phaseName.In(ClusterAddonsPhaseNames):
            handleClusterAddonsPhase(ctx, err)
            
        case phaseName.In(ClusterUpgradePhaseNames):
            handleClusterUpgradePhase(ctx, err)
            
        case phaseName.In(ClusterScaleMasterDownPhaseNames):
            handleClusterScaleMasterDownPhase(ctx, err)
            
        case phaseName.In(ClusterScaleWorkerDownPhaseNames):
            handleClusterScaleWorkerDownPhase(ctx, err)
            
        case phaseName.In(ClusterManagePhaseNames):
            handleClusterManagePhase(ctx, err)
            
        default:
            ctx.BKECluster.Status.ClusterStatus = ClusterUnknown
    }
}

func (b *StatusManager) recordBKEClusterStatus(bkeCluster *BKECluster) {
    // Step 1: 检查是否需要记录
    if !HasAnnotation(bkeCluster, StatusRecordAnnotationKey) {
        return
    }
    
    // Step 2: 获取状态记录
    sr := b.BKEClusterStatusMap[key]
    
    // Step 3: 不记录暂停状态
    if state == ClusterPaused {
        sr.NeedRequeue = false
        return
    }
    
    // Step 4: 设置当前集群健康状态
    sr.SetCurrentClusterState(bkeCluster.Status.ClusterHealthState)
    
    // Step 5: 判断是否为失败状态
    failedState := strings.HasSuffix(state, "Failed")
    
    // Step 6: 正常状态处理
    if !failedState {
        sr.SetLatestNormalState(state)
        sr.NeedRequeue = false
        return
    }
    
    // Step 7: 失败状态处理
    if sr.Equal(state) {
        sr.Inc()  // 与上次一致，计数器+1
    } else {
        sr.Reset()  // 与上次不一致，重置计数器
        sr.SetLatestFailedState(state)
        sr.Inc()
    }
    
    // Step 8: 判断是否允许重试
    if sr.AllowFailed() {
        // 未超过限制，恢复到上一次正常状态
        bkeCluster.Status.ClusterStatus = sr.LatestNormalState
        sr.NeedRequeue = true
    } else {
        // 超过限制，停止重试
        // 设置集群健康状态为Failed
        sr.NeedRequeue = false
    }
}
```

### 8.4 状态转换流程图

```
Phase执行
    ↓
设置StatusRecordAnnotationKey注解
    ↓
执行Phase逻辑
    ↓
Phase执行结果
    ├─ 成功 → 设置正常状态
    │          ├─ 记录LatestNormalState
    │          └─ NeedRequeue = false
    │
    └─ 失败 → 设置失败状态
               ├─ 判断是否与上次失败状态一致
               │   ├─ 一致 → StatusCount++
               │   └─ 不一致 → Reset() + StatusCount = 1
               │
               └─ 判断是否允许重试
                   ├─ StatusCount < AllowedFailedCount
                   │   ├─ 恢复到LatestNormalState
                   │   └─ NeedRequeue = true
                   │
                   └─ StatusCount >= AllowedFailedCount
                       ├─ 设置ClusterHealthState为Failed
                       └─ NeedRequeue = false
```

## 11. 状态机关键特性总结

### 11.1 状态隔离

- **集群状态**: 管理集群整体生命周期
- **Phase状态**: 管理Phase执行状态
- **节点状态**: 管理单个节点生命周期
- **健康状态**: 管理集群健康检查结果

### 11.2 状态转换规则

- **单向转换**: 大部分状态转换是单向的
- **失败回退**: 失败状态可以回退到正常状态（通过重试）
- **状态保护**: 特殊状态（暂停、删除）有保护机制

### 11.3 失败处理机制

- **计数器机制**: 记录连续失败次数
- **自动重试**: 未超过限制时自动重试
- **状态恢复**: 重试时恢复到上一次正常状态
- **停止重试**: 超过限制后停止重试，避免无限循环

### 11.4 状态监控

- **实时监控**: WatchBKEClusterStatus监控状态变更
- **状态记录**: StatusRecordAnnotationKey标记需要记录的状态
- **历史清理**: 最多保留MaxPhaseStatusHistory个Phase状态

### 11.5 并发安全

- **读写锁**: cmux和nmux保护状态Map
- **原子操作**: StatusCount等计数器原子递增
- **状态隔离**: 集群状态和节点状态独立管理
