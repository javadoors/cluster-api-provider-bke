## 一、总则

### 1.1 目的

本规范旨在建立统一的升级过程异常处理标准，确保：
1. 异常场景识别完整、分类清晰
2. 异常处理策略一致、可预测
3. 重试机制合理、可控
4. 日志记录规范、可追溯
5. 故障恢复自动化、可人工介入
6. 升级失败可回滚、数据不丢失

### 1.2 适用范围

本规范适用于升级流程的以下Phase，按执行顺序排列：

**PostDeployPhases (10个，集群升级时执行)**
- EnsureProviderSelfUpgrade（Provider自升级）
- EnsureAgentUpgrade（Agent升级）
- EnsureContainerdUpgrade（Containerd升级）
- EnsureEtcdUpgrade（Etcd升级）
- EnsureWorkerUpgrade（Worker升级）
- EnsureMasterUpgrade（Master升级）
- EnsureWorkerDelete（Worker删除）
- EnsureMasterDelete（Master删除）
- EnsureComponentUpgrade（核心组件升级）
- EnsureCluster（集群健康检查）

### 1.3 规范等级

| 等级 | 标识 | 说明 | 违反后果 |
|------|------|------|---------|
| **强制** | [MUST] | 必须遵守，无例外 | 代码审核不通过 |
| **推荐** | [SHOULD] | 强烈建议遵守，特殊情况可例外 | 需说明理由 |
| **可选** | [MAY] | 可选择遵守 | 无 |

### 1.4 术语定义

| 术语 | 定义 |
|------|------|
| **Fail-Fast** | 遇到错误立即中断流程，不自动重试，需人工介入 |
| **Best-Effort** | 遇到错误记录失败，继续处理其他资源，最后统一报告 |
| **Requeue** | Controller重新将请求加入工作队列，触发再次协调 |
| **BackoffLimit** | 命令执行失败时的最大重试次数 |
| **ActiveDeadlineSecond** | 命令执行的最大允许时间（秒） |
| **Phase** | 升级流程中的一个逻辑阶段 |
| **Condition** | 命令执行状态的详细记录 |
| **Rollback** | 升级失败后回滚到上一稳定版本的操作 |
| **Drain** | 驱逐节点上的Pod，为升级或删除做准备 |
| **Uncordon** | 恢复节点调度，允许Pod重新调度到该节点 |

## 二、异常分类规范

### 2.1 异常严重程度分类

**[MUST]** 所有异常必须按照以下标准分类：

| 严重程度 | 定义 | 处理原则 | 日志级别 | 事件类型 |
|---------|------|---------|---------|---------|
| **Fatal** | 导致升级流程无法继续，需人工介入 | Fail-Fast，立即中断 | Error | Warning |
| **Error** | 当前操作失败，但可重试或跳过 | 根据策略重试或标记失败 | Error | Warning |
| **Warning** | 非预期情况，但不影响主流程 | 记录日志，继续执行 | Warn | Normal |
| **Info** | 正常流程中的关键节点 | 记录日志，继续执行 | Info | Normal |

**[MUST]** Fatal级别异常必须满足以下条件之一：
1. Etcd备份失败或Etcd集群不可用
2. Master节点升级失败
3. API Server不可达
4. Etcd成员移除失败（Master删除场景）
5. 关键配置错误导致无法继续

### 2.2 异常来源分类

**[MUST]** 异常必须明确来源分类：

| 来源 | 说明 | 典型异常 | 默认处理策略 |
|------|------|---------|-------------|
| **Infrastructure** | K8s API、网络、存储等底层异常 | API Server不可达、Secret不存在 | Requeue重试 |
| **Node** | 节点层面异常 | 节点NotReady、Drain失败、节点删除失败 | 单节点重试 |
| **Cluster** | 集群层面异常 | Etcd不健康、控制平面不可用 | 等待轮询或Fail-Fast |
| **Configuration** | 配置错误或缺失 | 版本不兼容、升级路径被阻止 | 返回错误，需人工修正 |
| **Command** | Agent命令执行失败 | Kubeadm upgrade失败、脚本执行错误 | 根据BackoffLimit重试 |
| **Version** | 版本相关异常 | 版本回退失败、版本不一致 | 记录警告或返回错误 |

### 2.3 异常场景完整性要求

**[MUST]** 每个Phase必须识别并文档化以下异常场景：
1. 前置条件不满足的场景（如集群不健康、版本不兼容）
2. 资源获取失败的场景（如远程客户端获取失败）
3. 核心逻辑执行失败的场景（如升级命令执行失败）
4. 状态持久化失败的场景（如节点状态更新失败）
5. 超时场景（如健康检查超时、命令执行超时）
6. 配置错误场景（如版本不匹配、镜像不存在）
7. 回滚失败场景（升级失败后无法回滚）

**[SHOULD]** 每个异常场景应包含：
- 触发条件
- 严重程度
- 处理策略
- 重试机制
- 日志输出
- 人工介入条件
- 回滚方案（如适用）

## 三、异常处理策略规范

### 3.1 Phase处理策略选择

**[MUST]** Phase处理策略必须按照以下规则选择：

```
┌─────────────────────────────────────────────────────────────┐
│  Phase处理策略选择规则                                       │
└─────────────────────────────────────────────────────────────┘

IF Phase IN {EnsureEtcdUpgrade} THEN
    策略 = Fail-Fast
    理由 = Etcd是集群数据存储核心，备份失败或集群不健康必须人工介入
    
ELSE IF Phase IN {EnsureMasterUpgrade} THEN
    策略 = Fail-Fast
    理由 = Master是控制平面核心，升级失败必须人工介入
    
ELSE IF Phase IN {EnsureMasterDelete} THEN
    策略 = Fail-Fast (Etcd成员移除) + Per-Node-Retry (节点删除)
    理由 = Etcd成员移除失败必须人工介入，节点删除可重试
    
ELSE IF Phase IN {EnsureWorkerUpgrade, EnsureWorkerDelete} THEN
    策略 = Best-Effort
    理由 = Worker失败不影响集群核心功能
    
ELSE IF Phase IN {EnsureProviderSelfUpgrade, EnsureAgentUpgrade,
                  EnsureComponentUpgrade, EnsureCluster} THEN
    策略 = Requeue-Until-Success
    理由 = 临时性错误，可自动恢复
    
ELSE IF Phase IN {EnsureContainerdUpgrade} THEN
    策略 = Per-Node-Retry
    理由 = 单节点失败不影响其他节点
END IF
```

### 3.2 Fail-Fast策略规范

**[MUST]** 采用Fail-Fast策略时必须遵守：

```go
// 标准Fail-Fast处理模板
func (e *Phase) Execute() (ctrl.Result, error) {
    // 执行操作
    err := e.doOperation()
    
    if err != nil {
        // [MUST] 立即标记失败状态
        e.markFailed(err)
        
        // [MUST] 持久化状态
        if syncErr := e.syncStatus(); syncErr != nil {
            return ctrl.Result{}, syncErr
        }
        
        // [MUST] 记录Error级别日志
        log.Error(constant.PhaseFailedReason, 
            "Phase failed, need manual intervention: %v", err)
        
        // [MUST] 返回错误，不Requeue
        return ctrl.Result{}, err
    }
    
    return ctrl.Result{}, nil
}
```

**[MUST]** Fail-Fast场景下：
1. 不设置BackoffLimit（或设置为0）
2. 不执行自动重试
3. 必须记录人工介入指引日志
4. 必须标记节点/集群状态为Failed
5. **[MUST]** Etcd升级前必须确认备份成功，否则Fail-Fast

### 3.3 Best-Effort策略规范

**[MUST]** 采用Best-Effort策略时必须遵守：

```go
// 标准Best-Effort处理模板
func (e *Phase) Execute() (ctrl.Result, error) {
    var failedResources []string
    
    for _, resource := range resources {
        // 执行操作
        err := e.doOperation(resource)
        
        if err != nil {
            // [MUST] 记录失败资源
            failedResources = append(failedResources, resource.ID)
            
            // [MUST] 标记单个资源失败
            e.markResourceFailed(resource, err)
            
            // [MUST] 记录Warning级别日志
            log.Warn(constant.ResourceFailedReason,
                "Resource %s failed: %v, continuing with others", 
                resource.ID, err)
            
            // [MUST] 继续处理下一个资源
            continue
        }
        
        // 标记成功
        e.markResourceSuccess(resource)
    }
    
    // [MUST] 持久化所有状态
    if err := e.syncStatus(); err != nil {
        return ctrl.Result{}, err
    }
    
    // [MUST] 如有失败资源，返回错误触发Requeue
    if len(failedResources) > 0 {
        return ctrl.Result{}, errors.Errorf(
            "some resources failed: %v", failedResources)
    }
    
    return ctrl.Result{}, nil
}
```

### 3.4 Requeue策略规范

**[MUST]** Requeue策略必须明确区分场景：

```go
// Requeue决策规范
func (e *Phase) handleError(err error) (ctrl.Result, error) {
    // [MUST] 资源不存在不Requeue
    if apierrors.IsNotFound(err) {
        return ctrl.Result{}, nil
    }
    
    // [MUST] 版本冲突立即Requeue
    if apierrors.IsConflict(err) {
        return ctrl.Result{Requeue: true}, nil
    }
    
    // [MUST] 临时性错误延迟Requeue
    if isTemporaryError(err) {
        return ctrl.Result{RequeueAfter: quickRequeueInterval}, nil
    }
    
    // [MUST] 其他错误返回错误
    return ctrl.Result{}, err
}
```

**[SHOULD]** Requeue间隔应遵循：
- 快速重试：10秒（临时性错误）
- 中速重试：30秒（资源暂时不可用）
- 慢速重试：5分钟（周期性检查）

### 3.5 升级前预检规范

**[MUST]** 升级流程执行前必须进行预检：

```go
// 升级前预检规范
func (e *Phase) preUpgradeCheck() error {
    // [MUST] 1. 检查集群状态是否健康
    if bkeCluster.Status.ClusterStatus == bkev1beta1.ClusterUnhealthy ||
        bkeCluster.Status.ClusterStatus == bkev1beta1.ClusterUnknown {
        return errors.Errorf("cluster is not healthy, status: %s", 
            bkeCluster.Status.ClusterStatus)
    }
    
    // [MUST] 2. 检查所有节点是否Ready
    nodes, err := e.getAllNodes()
    if err != nil {
        return err
    }
    for _, node := range nodes {
        if !e.isNodeReady(node) {
            return errors.Errorf("node %s is not ready", node.Name)
        }
    }
    
    // [MUST] 3. 检查Etcd集群是否健康
    if err := e.checkEtcdClusterHealth(); err != nil {
        return errors.Errorf("etcd cluster is not healthy: %v", err)
    }
    
    // [MUST] 4. 检查版本兼容性
    if err := e.checkVersionCompatibility(); err != nil {
        return errors.Errorf("version compatibility check failed: %v", err)
    }
    
    // [MUST] 5. 检查资源是否充足（磁盘、内存）
    if err := e.checkResourceAvailability(); err != nil {
        return errors.Errorf("resource availability check failed: %v", err)
    }
    
    return nil
}
```

### 3.6 升级后验证规范

**[MUST]** 升级流程完成后必须进行验证：

```go
// 升级后验证规范
func (e *Phase) postUpgradeValidation() error {
    // [MUST] 1. 验证所有节点版本
    nodes, err := e.getAllNodes()
    if err != nil {
        return err
    }
    for _, node := range nodes {
        if node.Status.NodeInfo.KubeletVersion != targetVersion {
            return errors.Errorf("node %s version mismatch: expected %s, got %s", 
                node.Name, targetVersion, node.Status.NodeInfo.KubeletVersion)
        }
    }
    
    // [MUST] 2. 验证Etcd集群健康
    if err := e.checkEtcdClusterHealth(); err != nil {
        return errors.Errorf("etcd cluster health check failed: %v", err)
    }
    
    // [MUST] 3. 验证API Server可用性
    if err := e.checkAPIServerHealth(); err != nil {
        return errors.Errorf("API Server health check failed: %v", err)
    }
    
    // [MUST] 4. 验证组件版本
    if err := e.checkComponentVersions(); err != nil {
        return errors.Errorf("component version check failed: %v", err)
    }
    
    // [MUST] 5. 验证集群功能
    if err := e.validateClusterFunctionality(); err != nil {
        return errors.Errorf("cluster functionality validation failed: %v", err)
    }
    
    return nil
}
```

## 四、重试机制规范

### 4.1 重试配置规范

**[MUST]** Command CRD重试配置必须明确设置：

```yaml
spec:
  # [MUST] 明确设置最大重试次数
  backoffLimit: <n>  # n >= 0
  
  # [MUST] 明确设置执行超时
  activeDeadlineSecond: <t>  # t > 0
  
  # [SHOULD] 设置完成后保留时间
  ttlSecondsAfterFinished: <ttl>  # ttl >= 0
  
  commands:
    - id: "<unique-id>"
      # [MUST] 每个命令明确是否跳过失败
      backoffIgnore: <true|false>
      
      # [SHOULD] 设置重试间隔
      backoffDelay: <d>  # d >= 0
```

### 4.2 重试次数规范

**[MUST]** 重试次数必须根据场景合理设置：

| 场景 | BackoffLimit | 理由 |
|------|-------------|------|
| **Etcd升级** | 0 | Fail-Fast，不重试 |
| **Master升级** | 0 | Fail-Fast，不重试 |
| **Master删除 (Etcd成员移除)** | 0 | Fail-Fast，不重试 |
| **Master删除 (节点删除)** | 3 | 允许有限重试 |
| **Worker升级** | 3 | 允许有限重试 |
| **Worker删除** | 3 | 允许有限重试 |
| **Containerd升级** | 3 | 允许有限重试 |
| **Provider自升级** | 0 | 自动重试，无限制 |
| **Agent升级** | 0 | 自动重试，无限制 |
| **组件升级** | 0 | 自动重试，无限制 |
| **集群健康检查** | 0 | 自动重试，无限制 |

**[MUST]** 重试次数不得超过5次，超过需人工介入。

### 4.3 重试间隔规范

**[MUST]** 重试间隔必须合理设置：

| 场景 | BackoffDelay | 理由 |
|------|-------------|------|
| **网络操作** | 5-10秒 | 等待网络恢复 |
| **服务启动** | 3-5秒 | 等待服务就绪 |
| **资源创建** | 2-3秒 | 快速重试 |
| **配置生效** | 1-2秒 | 配置快速生效 |
| **健康检查** | 2-10秒 | 等待组件就绪 |

**[SHOULD]** 重试间隔应考虑：
- 操作类型（网络、磁盘、CPU）
- 资源规模（单节点、多节点）
- 环境状况（生产、测试）

### 4.4 超时配置规范

**[MUST]** 超时时间必须根据操作复杂度设置：

| 操作类型 | ActiveDeadlineSecond | 计算依据 |
|---------|---------------------|---------|
| **Provider自升级** | 300（5分钟） | Deployment更新+Pod重启 |
| **Agent升级** | 600（10分钟） | DaemonSet滚动更新+Pod就绪 |
| **Containerd升级** | 600（10分钟） | 重置配置+重新部署 |
| **Etcd升级** | 900（15分钟） | 备份+滚动升级+健康检查 |
| **Worker升级** | 600-1200（10-20分钟） | Drain+Kubeadm upgrade+Uncordon |
| **Master升级** | 900（15分钟） | Kubeadm upgrade+组件重启 |
| **Worker删除** | 600（10分钟） | Drain+节点删除 |
| **Master删除** | 900（15分钟） | Etcd成员移除+Drain+节点删除 |
| **组件升级** | 600（10分钟） | 镜像更新+Pod重启 |
| **集群健康检查** | 300（5分钟） | 全面健康检查 |

**[MUST]** 超时时间计算公式：

```
ActiveDeadlineSecond = 基础时间 + (节点数 × 单节点时间) + 网络延迟补偿

其中：
- 基础时间：操作本身耗时（如Kubeadm upgrade约2分钟）
- 单节点时间：每个节点额外耗时（如Drain+Uncordon约2分钟）
- 网络延迟补偿：离线环境增加50%，跨机房增加100%
```

### 4.5 重试终止条件

**[MUST]** 重试必须在以下条件之一满足时终止：

```go
func (c *Command) shouldStopRetry() bool {
    // [MUST] 达到最大重试次数
    if c.retryCount >= c.spec.BackoffLimit {
        return true
    }
    
    // [MUST] 达到执行超时
    if time.Since(c.startTime) > time.Duration(c.spec.ActiveDeadlineSecond)*time.Second {
        return true
    }
    
    // [MUST] 遇到不可重试错误
    if isUnrecoverableError(c.lastError) {
        return true
    }
    
    // [MUST] 资源被删除
    if c.isDeleted() {
        return true
    }
    
    return false
}
```

## 五、状态管理规范

### 5.1 节点状态管理规范

**[MUST]** 节点状态必须遵循状态机转换规则：

```
┌─────────────────────────────────────────────────────────────┐
│  节点状态转换图（升级场景）                                    │
└─────────────────────────────────────────────────────────────┘

Ready ──→ Upgrading ──→ Ready
   │           │
   │           └──→ UpgradeFailed
   │                    │
   │                    └──→ (NeedSkip=true 或 人工介入)
   │
   └──→ Deleting ──→ (Deleted)
            │
            └──→ DeleteFailed
                    │
                    └──→ (NeedSkip=true 或 人工介入)
```

**[MUST]** 状态转换必须满足：

```go
// 状态转换验证函数（升级场景）
func canTransitionTo(current, target NodeState) bool {
    // [MUST] 定义合法转换
    validTransitions := map[NodeState][]NodeState{
        NodeReady:       {NodeUpgrading, NodeDeleting, NodeNotReady},
        NodeUpgrading:   {NodeReady, NodeUpgradeFailed},
        NodeUpgradeFailed: {NodeUpgrading},  // 允许重试升级
        NodeDeleting:    {NodeDeleted, NodeDeleteFailed},
        NodeDeleteFailed: {NodeDeleting},    // 允许重试删除
        NodeNotReady:    {NodeReady, NodeUpgrading, NodeDeleting},
    }
    
    allowed, exists := validTransitions[current]
    if !exists {
        return false
    }
    
    for _, s := range allowed {
        if s == target {
            return true
        }
    }
    return false
}
```

### 5.2 状态持久化规范

**[MUST]** 状态变更必须立即持久化：

```go
// 标准状态更新模式
func (e *Phase) updateNodeState(nodeIP string, state NodeState, message string) error {
    // [MUST] 1. 检查状态转换合法性
    currentState := e.getNodeState(nodeIP)
    if !canTransitionTo(currentState, state) {
        return errors.Errorf("invalid state transition: %s -> %s", 
            currentState, state)
    }
    
    // [MUST] 2. 设置新状态
    e.setNodeState(nodeIP, state, message)
    
    // [MUST] 3. 立即持久化
    if err := mergecluster.SyncStatusUntilComplete(e.Client, e.BKECluster); err != nil {
        // [MUST] 4. 持久化失败必须返回错误
        return errors.Wrap(err, "failed to sync node state")
    }
    
    // [MUST] 5. 记录状态变更日志
    log.Info("Node state changed: %s -> %s, message: %s", 
        currentState, state, message)
    
    return nil
}
```

**[MUST]** 状态持久化失败处理：
1. 必须返回错误，触发Requeue
2. 必须记录Error级别日志
3. 不得继续执行后续操作

### 5.3 Phase状态管理规范

**[MUST]** Phase状态必须通过ExecutePreHook/ExecutePostHook自动管理：

```go
func (e *Phase) ExecutePreHook() error {
    // [MUST] 自动设置Phase为Running
    e.SetStatus(bkev1beta1.PhaseRunning)
    return e.BasePhase.DefaultPreHook()
}

func (e *Phase) ExecutePostHook(execErr error) error {
    // [MUST] 根据执行结果设置Phase状态
    if execErr == nil {
        e.SetStatus(bkev1beta1.PhaseSucceeded)
    } else {
        e.SetStatus(bkev1beta1.PhaseFailed)
    }
    return e.BasePhase.DefaultPostHook(execErr)
}
```

### 5.4 升级失败回滚规范

**[MUST]** 升级失败时必须支持回滚：

```go
// 标准回滚处理模板
func (e *Phase) rollback() error {
    // [MUST] 1. 保存当前状态用于回滚
    currentState := e.saveCurrentState()
    
    // [MUST] 2. 执行回滚操作
    err := e.doRollback(currentState)
    if err != nil {
        // [MUST] 3. 回滚失败必须记录Error日志
        log.Error(constant.RollbackFailedReason,
            "Rollback failed: %v", err)
        return err
    }
    
    // [MUST] 4. 回滚成功后更新状态
    e.markRolledBack()
    if syncErr := e.syncStatus(); syncErr != nil {
        return syncErr
    }
    
    log.Info("Rollback completed successfully")
    return nil
}
```

**[MUST]** 回滚场景：
1. Etcd升级失败且备份存在 → 回滚到备份版本
2. Master升级失败 → 需人工介入，手动回滚
3. Worker升级失败 → 可重新升级该节点
4. 组件升级失败 → 回滚到旧版本镜像
5. Agent升级失败 → 回滚到旧版本DaemonSet

## 六、日志记录规范

### 6.1 日志级别使用规范

**[MUST]** 日志级别必须按照以下标准使用：

| 级别 | 使用场景 | 示例 |
|------|---------|------|
| **Info** | 正常流程关键步骤、状态变更、进度信息 | `log.Info("Master node upgrade started")` |
| **Warn** | 可恢复异常、非预期情况、降级处理 | `log.Warn("Worker node drain failed, force drain")` |
| **Error** | 严重错误、流程中断、需人工介入 | `log.Error("Etcd upgrade failed: %v", err)` |
| **Debug** | 调试信息、详细执行过程（生产环境禁用） | `log.Debug("Poll count: %d", count)` |

**[MUST]** 日志输出频率控制：

```go
// [MUST] 轮询场景必须控制日志频率

        log.Info("Polling status, count: %d", i)
    }
}
```

### 6.2 日志内容规范

**[MUST]** 日志必须包含以下要素：

```go
// 标准日志格式
log.<Level>(
```

**[MUST]** 日志内容要求：
1. 必须包含事件Reason（用于事件记录）
2. 必须包含人类可读的消息

## 七、代码实现规范

### 7.1 标准Phase实现模板

**[MUST]** 所有Phase必须遵循以下实现模板：

```go
// 文件: pkg/phaseframe/phases/ensure_xxx_upgrade.go

// [MUST] 1. 定义Phase名称常量
const (
    EnsureXxxUpgradeName confv1beta1.BKEClusterPhase = "EnsureXxxUpgrade"

// [MUST] 2. 定义Phase结构体

    // [SHOULD] 添加Phase特定字段
}

// [MUST] 3. 实现构造函数
func NewEnsureXxxUpgrade(ctx *phaseframe.PhaseContext) phaseframe.Phase {
    base := phaseframe.NewBasePhase(ctx, EnsureXxxUpgradeName)
    return &EnsureXxxUpgrade{BasePhase: base}
}

// [MUST] 4. 实现NeedExecute方法
func (e *EnsureXxxUpgrade) NeedExecute(old, new *bkev1beta1.BKECluster) bool {
    // [MUST] 调用默认检查
    if !e.BasePhase.DefaultNeedExecute(old, new) {
        return false
    }
    
    // [MUST] 添加Phase特定检查
    needExecute := e.checkSpecificCondition()
    
    // [MUST] 设置Phase状态
    if needExecute {
        e.SetStatus(bkev1beta1.PhaseWaiting)
    }
    
    return needExecute
}

// [MUST] 5. 实现Execute方法
func (e *EnsureXxxUpgrade) Execute() (ctrl.Result, error) {
    ctx, c, bkeCluster, _, log := e.Ctx.Untie()
    
    // [MUST] 5.1 前置检查
    if !e.preCheck() {
        log.Info("precondition not met")
        return ctrl.Result{RequeueAfter: quickRequeueInterval}, nil
    }
    
    // [MUST] 5.2 获取资源
    resources, err := e.getResources()
    if err != nil {
        // [MUST] 区分错误类型
        if apierrors.IsNotFound(err) {
            return ctrl.Result{}, nil
        }
        log.Error("failed to get resources: %v", err)
        return ctrl.Result{}, err
    }
    
    // [MUST] 5.3 执行核心逻辑（根据策略选择模板）
    result, err := e.doExecute(resources)
    if err != nil {
        return result, err
    }
    
    // [MUST] 5.4 后置处理
    if err := e.postProcess(); err != nil {
        log.Warn("post process failed: %v", err)
    }
    
    return ctrl.Result{}, nil
}

// [MUST] 6. 实现PreHook/PostHook（可选）
func (e *EnsureXxxUpgrade) ExecutePreHook() error {
    return e.BasePhase.DefaultPreHook()
}
```

### 7.2 错误处理规范

### 7.4 升级特定规范

**[MUST]** 升级Phase必须实现版本比对：

```go
// 版本比对规范
func (e *Phase) needUpgrade(currentVersion, targetVersion string) bool {
    // [MUST] 版本相同不需要升级
    if currentVersion == targetVersion {
        return false
    }
    
    // [MUST] 检查版本兼容性
    if !e.isVersionCompatible(currentVersion, targetVersion) {
        log.Error("Version %s is not compatible with %s", 
            currentVersion, targetVersion)
        return false
    }
    
    return true
}
```

**[MUST]** 升级Phase必须支持Drain/Uncordon：

```go
// Drain规范
func (e *Phase) drainNode(node Node) error {
    // [MUST] 1. 标记节点为不可调度
    if err := e.cordonNode(node); err != nil {
        return errors.Wrap(err, "failed to cordon node")
    }
    
    // [MUST] 2. 驱逐Pod
    if err := e.evictPods(node); err != nil {
        // [SHOULD] 驱逐失败尝试强制驱逐
        log.Warn("Pod eviction failed, attempting force drain: %v", err)
        if forceErr := e.forceDrainNode(node); forceErr != nil {
            return errors.Wrap(forceErr, "failed to force drain node")
        }
    }
    
    return nil
}

// Uncordon规范
func (e *Phase) uncordonNode(node Node) error {
    // [MUST] 恢复节点调度
    if err := e.uncordonNode(node); err != nil {
        // [MUST] 记录错误但不阻断流程
        log.Error("Failed to uncordon node %s: %v", node.IP, err)
        return err
    }
    
    return nil
}
```
