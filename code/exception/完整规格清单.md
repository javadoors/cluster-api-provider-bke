### 2.6 安装Phase详细处理流程

#### 2.6.1 EnsureCerts 处理流程

**Phase名称**: EnsureCerts  
**Phase功能**: 生成和管理集群证书  
**触发条件**: 集群创建时或证书即将过期  
**失败策略**: Requeue重试（无限重试）  
**代码位置**: [ensure_certs.go](file:///c:/Users/z00820145/code/github/cluster-api-provider-bke/pkg/phaseframe/phases/ensure_certs.go)

**前置条件检查**:
1. BKECluster对象存在且有效
2. 节点列表已获取
3. 证书生成器初始化成功

**主要执行步骤**:
```
Step 1: 初始化证书生成器
  ├─ 操作: 创建BKEKubernetesCertGenerator实例
  ├─ 输入: BKECluster对象、节点列表
  ├─ 输出: certsGenerator实例
  └─ 异常: 初始化失败返回错误

Step 2: 查找或生成证书
  ├─ 操作: 调用LookUpOrGenerate()方法
  ├─ 输入: certsGenerator实例
  ├─ 输出: 证书Secret对象
  └─ 异常: 
      - Secret查找失败 → Requeue重试
      - 证书生成失败 → Requeue重试
      - Secret存储失败 → Requeue重试

Step 3: 检查是否需要重新生成
  ├─ 操作: 调用NeedGenerate()检查证书有效性
  ├─ 输入: 证书Secret对象
  ├─ 输出: need=true/false
  └─ 异常: 
      - need=true → 返回错误触发重新生成
      - 检查失败 → 返回错误
```

**Phase名称**: EnsureCluster

3. 集群未处于特殊状态（扩缩容/升级/暂停/初始化）
4. 后置处理已完成

**主要执行步骤**:
```
Step 1: 检查Cluster对象
  ├─ 操作: 验证Cluster对象存在
  ├─ 输入: BKECluster.OwnerReferences
  ├─ 输出: Cluster对象
  └─ 异常:

Step 2: 检查控制平面初始化状态
  ├─ 操作: 检查ControlPlaneInitializedCondition
  ├─ 输入: Cluster.Status.Conditions
  ├─ 输出: 控制平面已初始化
  └─ 异常:
      - Condition不为True → 返回错误

Step 3: 获取远程集群客户端
  ├─ 操作: 连接到目标K8s集群
  ├─ 输入: BKECluster配置
  ├─ 输出: RemoteKubeClient
  └─ 异常:
      - 连接失败 → 返回错误

Step 4: 设置告警标签（BKE集群）
  ├─ 操作: 为BKE集群设置告警标签
  ├─ 输入: RemoteKubeClient
  ├─ 输出: 告警标签设置
  └─ 处理逻辑:
      - 查询已有告警标签的节点
      - 如果不存在 → 选择一个Worker节点
      - 设置alert label
  └─ 异常:
      - 查询失败 → 记录错误继续
      - 设置失败 → 记录错误继续

Step 5: 设置裸金属标签（Richrunc集群）
  ├─ 操作: 为裸金属集群设置标签
  ├─ 输入: 
      - RemoteKubeClient
      - ContainerRuntime配置
  ├─ 输出: 裸金属标签设置
  └─ 判断:
      - ContainerRuntime.Runtime == "richrunc" → 设置标签
  └─ 异常:
      - 设置失败 → 记录错误继续

Step 6: 设置节点标签
  ├─ 操作: 为所有节点设置角色标签
  ├─ 输入: 
      - RemoteKubeClient
      - BKENode列表
  ├─ 输出: 节点标签设置
  └─ 处理逻辑:
      - 获取所有节点
      - 根据BKENode角色设置标签
      - Master节点 → node-role.kubernetes.io/control-plane
      - Worker节点 → node-role.kubernetes.io/node
  └─ 异常:
      - 设置失败 → 记录错误继续

Step 7: 确保K8s Token有效
  ├─ 操作: 检查并更新Bootstrap Token
  ├─ 输入: RemoteKubeClient
  ├─ 输出: Token有效
  └─ 处理逻辑:
      - 查询现有Token列表
      - 检查Token是否过期
      - 如果过期 → 创建新Token
      - 更新Token Secret
  └─ 异常:
      - 查询失败 → 记录错误继续
      - 创建失败 → 记录错误继续

Step 8: 检查集群特殊状态
  ├─ 操作: 判断集群是否处于特殊状态
  ├─ 输入: BKECluster.Status.ClusterStatus
  ├─ 输出: 是否跳过健康检查
  └─ 特殊状态列表:
      - ClusterMasterScalingUp
      - ClusterMasterScalingDown
      - ClusterWorkerScalingUp
      - ClusterWorkerScalingDown
      - ClusterInitializing
      - ClusterPaused
      - ClusterUpgrading
  └─ 判断:
      - 处于特殊状态 → 返回错误跳过检查
      - 正常状态 → 继续健康检查

Step 9: 检查后置处理状态
  ├─ 操作: 检查节点后置处理是否完成
  ├─ 输入: BKENode列表
  ├─ 输出: 后置处理完成
  └─ 处理逻辑:
      - 获取需要后置处理的节点
      - 如果存在未完成节点 → 设置Condition并Requeue
  └─ 异常:
      - 存在未完成节点 → 返回错误快速Requeue

Step 10: 确保集群Ready
  ├─ 操作: 执行集群健康检查
  ├─ 输入: 
      - RemoteKubeClient
      - BKECluster
  ├─ 输出: 集群健康状态
  └─ 检查内容:
      - 所有节点Ready状态
      - 控制平面Pod运行状态
      - Etcd健康状态
      - 关键组件状态
  └─ 异常:
      - 检查失败 → 返回错误快速Requeue
      - 集群不健康 → 返回错误快速Requeue

Step 11: 返回定期Requeue
  ├─ 操作: 设置下次检查时间
  ├─ 输入: 无
  ├─ 输出: RequeueAfter=5分钟
  └─ 目的:
      - 定期检查集群健康
      - 及时发现异常状态
```

| 后置处理未完成 | 快速Requeue | 无限 | 否 |
