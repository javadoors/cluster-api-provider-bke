# BKE 分阶段组件版本升级方案设计
## 一、整体设计思路
### 1.1 核心理念：声明式组件生命周期管理
```
┌─────────────────────────────────────────────────────────────────────────┐
│                         设计核心原则                                    │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  1. 控制器 → 调和器                                                     │
│     ├── 控制器只负责状态驱动与容错重试                                  │
│     └── 无具体组件实现代码，所有逻辑声明在 YAML 中                      │
│                                                                         │
│  2. 组件 → 可插拔包                                                     │
│     ├── 每个组件独立版本化（K8s、etcd、containerd、CNI 等）             │
│     └── 提供统一的 LCM 接口（安装、卸载、升级、校验）                   │
│                                                                         │
│  3. Phase → 组件编排器                                                  │
│     ├── Phase 不关心组件细节，只负责调度与状态收集                      │
│     └── 组件的具体执行逻辑通过 Manifest 声明                            │
│                                                                         │
│  4. 升级 → 原子性变更                                                   │
│     ├── 版本对比 → 卸载旧版本 → 安装新版本 → 校验                       │
│     └── 支持回滚与故障检测                                              │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```
### 1.2 架构分层
```
┌─────────────────────────────────────────────────────────────────────────┐
│                         分层架构                                        │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  Layer 1: BKECluster Controller（调和层）                               │
│  ├── 监听 BKECluster 变化                                               │
│  ├── 触发 Phase 执行                                                    │
│  └── 收集状态与错误                                                     │
│                                                                         │
│  Layer 2: Phase Executor（编排层）                                      │
│  ├── 加载 ComponentVersion Manifest                                     │
│  ├── 比较版本差异（old vs new）                                         │
│  ├── 构建执行任务序列                                                   │
│  └── 监控执行进度                                                       │
│                                                                         │
│  Layer 3: Component Package��包声明层）                                 │
│  ├── ComponentVersion YAML：版本元信息                                  │
│  ├── LCM Manifest：安装/卸载/升级逻辑                                   │
│  ├── Compatibility Matrix：兼容性矩阵                                   │
│  └── Verification Spec：校验规则                                        │
│                                                                         │
│  Layer 4: Task Executor（执行层）                                       │
│  ├── builtin Plugin（内置插件执行）                                     │
│  ├── k8s Resource（K8s 资源操作）                                       │
│  └── shell Script（Shell 脚本执行）                                     │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```
### 1.3 数据流
```
BKECluster (spec.version)
    │
    ├─→ Load ComponentVersion Manifest
    │       └─→ k8s v1.28.0, etcd v3.5.9, containerd v1.7.2, calico v3.26
    │
    ├─→ Compare Old vs New
    │       ├─→ K8s: v1.27.0 → v1.28.0 (upgrade)
    │       ├─→ etcd: v3.5.8 → v3.5.9 (patch upgrade)
    │       └─→ containerd: v1.7.1 → v1.7.2 (patch upgrade)
    │
    ├─→ Validate Compatibility
    │       ├─→ K8s v1.28.0 + etcd v3.5.9: ✓
    │       ├─→ K8s v1.28.0 + containerd v1.7.2: ✓
    │       └─→ calico v3.26 + K8s v1.28.0: ✓
    │
    ├─→ Generate Task Sequence
    │       ├─→ Uninstall K8s v1.27.0
    │       ├─→ Install K8s v1.28.0
    │       ├─→ Verify K8s v1.28.0
    │       ├─→ Drain & Upgrade etcd v3.5.8 → v3.5.9
    │       ├─→ Verify etcd cluster
    │       ├─→ Upgrade containerd v1.7.1 → v1.7.2
    │       ├─→ Verify containerd
    │       └─→ Update cluster status
    │
    └─→ Execute & Monitor
            └─→ State: Pending → InProgress → Succeeded/Failed
```

kind: ComponentVersion
metadata:
  name: k8s-v1.28.0
  namespace: bke-system
  labels:
    component: kubernetes
    version: "1.28.0"
    channel: stable
spec:
  # 基础信息
  component: kubernetes
  version: "1.28.0"
  
  # 版本信息
  versionInfo:
    kubeadmVersion: "1.28.0"
    kubeProxyVersion: "1.28.0"
    kubeControllerManagerVersion: "1.28.0"
    kubeApiserverVersion: "1.28.0"
    kubeSchedulerVersion: "1.28.0"
    cniVersion: "v1.3.0"  # CNI API version
    
  # 兼容性
  compatibility:
    # 依赖的其他组件版本范围
    dependencies:
      - component: containerd
        minVersion: "1.6.0"
        maxVersion: "2.0.0"
      - component: etcd
        minVersion: "3.5.0"
        maxVersion: "3.6.0"
      - component: cni
        minVersion: "1.2.0"
        maxVersion: "1.4.0"
    
    # 操作系统
    supportedOSes:
      - ubuntu-18.04
      - ubuntu-20.04
      - ubuntu-22.04
      - centos-7
      - centos-8
      
    # CPU 架构
    supportedArch:
      - amd64
      - arm64
  
  # 生命周期管理
  lifecycle:
    # 安装规格
    install:
      ref: install-k8s-v1.28.0  # 引用 LCM Manifest
      timeout: 300s
      retries: 3
      onFailure: Rollback
    
    # 卸载规格
    uninstall:
      ref: uninstall-k8s-v1.28.0
      timeout: 180s
      retries: 1
      onFailure: Log  # 只记录，不中断

      ref: downgrade-k8s-v1.28.0-to-v1.27.0
      supported: false  # 不支持从 1.28 降级到 1.27

    verify:
      ref: verify-k8s-v1.28.0
      timeout: 120s

      ref: healthcheck-k8s-v1.28.0

### 2.2 LCM Manifest（生命周期管理清单）
```yaml
---
# K8s 安装清单
apiVersion: component.bke.io/v1alpha1
kind: ComponentLifecycleManifest
metadata:
  name: install-k8s-v1.28.0
  namespace: bke-system
spec:
  component: kubernetes
  version: "1.28.0"
  operation: Install
  
  steps:
    # Step 1: 准备环境
    - name: prepare-environment
      type: shell
      description: "Prepare OS environment for Kubernetes v1.28.0"
      targets: [master, worker]
      script: |
        #!/bin/bash
        set -e
        
        # 检查 OS 兼容性
        . /etc/os-release
        case "$ID-$VERSION_ID" in
          ubuntu-20.04|ubuntu-22.04|centos-8|centos-9) ;;
          *) echo "Unsupported OS"; exit 1 ;;
        esac
        
        # 安装依赖
        if [ -f /etc/debian_version ]; then
          apt-get update
          apt-get install -y apt-transport-https ca-certificates curl
        else
          yum install -y curl ca-certificates
        fi
      
      onFailure: Fail
      retries: 3
    
    # Step 2: 下载二进制
    - name: download-binaries
      type: shell
      description: "Download Kubernetes v1.28.0 binaries"
      targets: [master, worker]
      env:
        K8S_VERSION: "1.28.0"
        DOWNLOAD_URL: "https://dl.k8s.io/release/v1.28.0/bin/linux"
      script: |
        #!/bin/bash
        set -e
        mkdir -p /opt/k8s/v1.28.0
        cd /opt/k8s/v1.28.0
        
        for bin in kubeadm kubelet kubectl; do
          curl -L "${DOWNLOAD_URL}/${ARCH}/${bin}" -o "${bin}"
          chmod +x "${bin}"
        done
      
      verification:
        type: shell
        script: |
          #!/bin/bash
          /opt/k8s/v1.28.0/kubelet --version | grep "v1.28.0"
          /opt/k8s/v1.28.0/kubeadm version | grep "v1.28.0"
      
      onFailure: Fail
      retries: 3
    
    # Step 3: 安装 kubelet systemd 服务
    - name: install-kubelet-service
      type: shell
      targets: [master, worker]
      script: |
        #!/bin/bash
        cat > /etc/systemd/system/kubelet.service << EOF
        [Unit]
        Description=kubelet: The Kubernetes Node Agent
        Documentation=https://kubernetes.io/docs/
        After=network.target
        
        [Service]
        ExecStart=/opt/k8s/v1.28.0/kubelet
        Restart=on-failure
        RestartSec=5
        
        [Install]
        WantedBy=multi-user.target
        EOF
        
        systemctl daemon-reload
        systemctl enable kubelet
      
      onFailure: Fail
    
    # Step 4: 初始化 Master（仅主 master 节点）
    - name: init-control-plane
      type: builtin
      plugin: kubeadm
      targets: [master-0]  # 仅第一个 master
      params:
        operation: init
        k8sVersion: "1.28.0"
        configPath: "/etc/kubernetes/kubeadm-config.yaml"
      
      onFailure: Fail
      retries: 1
    
    # Step 5: 等待 API Server 就绪
    - name: wait-apiserver
      type: builtin
      plugin: ping
      targets: [master-0]
      params:
        endpoint: "https://127.0.0.1:6443"
        timeout: 300s
        interval: 5s
      
      onFailure: Fail
      retries: 3
    
    # Step 6: 加入其他 master
    - name: join-control-planes
      type: builtin
      plugin: kubeadm
      targets: [master-1, master-2]  # 其他 master 节点
      parallelism: 1  # 串行加入，避免 etcd 冲突
      params:
        operation: join-control-plane
        k8sVersion: "1.28.0"
        tokenPath: "/opt/bootstrap/kubeadm-token"
      
      onFailure: Fail
      retries: 1
    
    # Step 7: 加入 worker 节点
    - name: join-workers
      type: builtin
      plugin: kubeadm
      targets: [worker]
      parallelism: 3  # 并行加入
      params:
        operation: join-worker
        k8sVersion: "1.28.0"
        tokenPath: "/opt/bootstrap/kubeadm-token"
      
      onFailure: Fail
      retries: 1
    
    # Step 8: 部署 CNI
    - name: deploy-cni
      type: builtin
      plugin: manifests
      targets: [master-0]
      params:
        manifestPath: "/opt/manifests/calico-v3.26.yaml"
      
      onFailure: Fail
    
    # Step 9: 等待集群就绪
    - name: verify-cluster-ready
      type: builtin
      plugin: healthcheck
      targets: [master-0]
      params:
        checksToRun: [nodes, apiserver, controller-manager, scheduler, cni]
        timeout: 600s
      
      onFailure: Fail
      retries: 3
  
  # 回滚策略
  rollback:
    enabled: true
    strategy: "reverse-order"  # 反向执行卸载
    onStepFailure: "automatic"  # 步骤失败自动回滚

---
# K8s 升级清单（版本特定）
apiVersion: component.bke.io/v1alpha1
kind: ComponentLifecycleManifest
metadata:
  name: upgrade-k8s-v1.27.0-to-v1.28.0
  namespace: bke-system
spec:
  component: kubernetes
  version: "1.28.0"
  sourceVersion: "1.27.0"
  operation: Upgrade
  
  steps:
    # Step 1: 检查升级前置条件
    - name: pre-upgrade-check
      type: shell
      targets: [master]
      script: |
        #!/bin/bash
        set -e
        
        # 检查当前版本
        CURRENT_VERSION=$(kubelet --version | grep -oP 'v\K[0-9.]+')
        if [ "$CURRENT_VERSION" != "1.27.0" ]; then
          echo "Expected v1.27.0, got $CURRENT_VERSION"
          exit 1
        fi
        
        # 检查集群健康状态
        kubectl get nodes -o json | jq -e '.items | all(.status.conditions[] | select(.type=="Ready") | .status == "True")' > /dev/null
        
        # 检查 etcd 集群状态
        kubectl exec -n kube-system etcd-* -- etcdctl member list
      
      onFailure: Fail
    
    # Step 2: 备份 etcd
    - name: backup-etcd
      type: builtin
      plugin: backup
      targets: [master-0]
      params:
        backupPath: "/opt/backups/etcd-pre-v1.28.0-upgrade"
        retentionDays: 30
      
      onFailure: Fail
    
    # Step 3: 更新 kubeadm 配置
    - name: update-kubeadm-config
      type: shell
      targets: [master]
      script: |
        #!/bin/bash
        set -e
        
        # 生成新的 kubeadm 配置
        /opt/k8s/v1.28.0/kubeadm config migrate --old-config /etc/kubernetes/kubeadm-config.yaml \
          > /etc/kubernetes/kubeadm-config-v1.28.0.yaml
      
      onFailure: Fail
    
    # Step 4: 升级 Master 节点（滚动）
    - name: upgrade-control-plane
      type: builtin
      plugin: kubeadm
      targets: [master]
      parallelism: 1  # 一次一个 master
      params:
        operation: upgrade-control-plane
        k8sVersion: "1.28.0"
        sourceVersion: "1.27.0"
        drainTimeout: 300s
      
      onFailure: Rollback
      retries: 1
    
    # Step 5: 等待 API Server 恢复
    - name: wait-apiserver-recover
      type: builtin
      plugin: ping
      targets: [master-0]
      params:
        endpoint: "https://127.0.0.1:6443"
        timeout: 300s
      
      onFailure: Fail
      retries: 5
    
    # Step 6: 升级 Worker 节点（并行，批次）
    - name: upgrade-workers
      type: builtin
      plugin: kubeadm
      targets: [worker]
      parallelism: 3  # 一次升级 3 个 worker
      batchSize: 5  # 每批 5 个
      params:
        operation: upgrade-worker
        k8sVersion: "1.28.0"
        sourceVersion: "1.27.0"
        drainTimeout: 300s
        evictionTimeout: 120s
      
      onFailure: SkipAndContinue  # worker 升级失败继续，master 优先
      retries: 1
    
    # Step 7: 验证升级结果
    - name: verify-upgrade
      type: shell
      targets: [master-0]
      script: |
        #!/bin/bash
        set -e
        
        # 等待所有节点就绪
        kubectl wait --for=condition=Ready nodes --all --timeout=300s
        
        # 检查版本
        KUBELET_VERSION=$(kubectl get nodes -o json | jq -r '.items[0].status.nodeInfo.kubeletVersion' | grep -oP 'v\K[0-9.]+')
        if [ "$KUBELET_VERSION" != "1.28.0" ]; then
          echo "Kubelet version mismatch: expected 1.28.0, got $KUBELET_VERSION"
          exit 1
        fi
        
        # 检查 API 版本
        API_VERSION=$(kubectl version --short | grep -oP 'Server Version: v\K[0-9.]+')
        if [ "$API_VERSION" != "1.28.0" ]; then
          echo "API version mismatch: expected 1.28.0, got $API_VERSION"
          exit 1
        fi
      
      onFailure: Fail
  
  rollback:
    enabled: true
    strategy: "restore-etcd"
    backupPath: "/opt/backups/etcd-pre-v1.28.0-upgrade"

---
# Etcd 升级清单（有状态）
apiVersion: component.bke.io/v1alpha1
kind: ComponentLifecycleManifest
metadata:
  name: upgrade-etcd-v3.5.8-to-v3.5.9
  namespace: bke-system
spec:
  component: etcd
  version: "3.5.9"
  sourceVersion: "3.5.8"
  operation: Upgrade
  
  # 有状态组件升级的关键特性
  statefulUpgrade: true
  
  steps:
    # Step 1: 前置检查
    - name: pre-upgrade-check
      type: shell
      targets: [etcd-0]
      script: |
        #!/bin/bash
        set -e
        
        export ETCDCTL_API=3
        ENDPOINT="https://127.0.0.1:2379"
        CERT="/etc/kubernetes/pki/etcd/server.crt"
        KEY="/etc/kubernetes/pki/etcd/server.key"
        CA="/etc/kubernetes/pki/etcd/ca.crt"
        
        # 检查集群健康
        etcdctl --endpoints=$ENDPOINT --cert=$CERT --key=$KEY --cacert=$CA member list
        etcdctl --endpoints=$ENDPOINT --cert=$CERT --key=$KEY --cacert=$CA endpoint health
      
      onFailure: Fail
    
    # Step 2: 快照备份（关键！）
    - name: snapshot-backup
      type: shell
      targets: [etcd-0]
      script: |
        #!/bin/bash
        set -e
        
        export ETCDCTL_API=3
        ENDPOINT="https://127.0.0.1:2379"
        CERT="/etc/kubernetes/pki/etcd/server.crt"
        KEY="/etc/kubernetes/pki/etcd/server.key"
        CA="/etc/kubernetes/pki/etcd/ca.crt"
        BACKUP_PATH="/opt/etcd-backup/snapshot-$(date +%s).db"
        
        mkdir -p /opt/etcd-backup
        etcdctl --endpoints=$ENDPOINT --cert=$CERT --key=$KEY --cacert=$CA \
          snapshot save "$BACKUP_PATH"
        
        # 验证快照
        etcdctl --write-out=table snapshot status "$BACKUP_PATH"
      
      onFailure: Fail
    
    # Step 3: 滚动升级（一个节点一个）
    - name: rolling-upgrade
      type: builtin
      plugin: etcd-upgrade
      targets: [etcd-0, etcd-1, etcd-2]
      parallelism: 1  # 严格串行
      params:
        sourceVersion: "3.5.8"
        targetVersion: "3.5.9"
        drainLeadership: true  # 升级前让出 leader
        healthCheckInterval: 10s
        healthCheckTimeout: 60s
      
      onFailure: Rollback
      retries: 1
    
    # Step 4: 升级后验证
    - name: post-upgrade-verify
      type: shell
      targets: [etcd-0]
      script: |
        #!/bin/bash
        set -e
        
        export ETCDCTL_API=3
        ENDPOINT="https://127.0.0.1:2379"
        CERT="/etc/kubernetes/pki/etcd/server.crt"
        KEY="/etc/kubernetes/pki/etcd/server.key"
        CA="/etc/kubernetes/pki/etcd/ca.crt"
        
        # 检查集群状态
        etcdctl --endpoints=$ENDPOINT --cert=$CERT --key=$KEY --cacert=$CA member list
        
        # 检查所有成员版本
        etcdctl --endpoints=$ENDPOINT --cert=$CERT --key=$KEY --cacert=$CA endpoint status \
          --write-out=json | jq '.[] | .Version'
      
      onFailure: Fail
  
  rollback:
    enabled: true
    strategy: "restore-snapshot"
    snapshotPath: "/opt/etcd-backup"
```

import (

const EnsureNodesEnvName = "EnsureNodesEnv"

type EnsureNodesEnv struct {

	componentManager *component.Manager

// NeedExecute：判断是否需要执行
func (e *EnsureNodesEnv) NeedExecute(old, new *bkev1beta1.BKECluster) bool {
	// 如果没有旧版本，肯定需要初始化
	if old == nil {
		return true
	}
	
	// 比较版本是否变化（通过组件包版本）
	oldManifests := e.componentManager.GetNodeEnvManifests(old)
	newManifests := e.componentManager.GetNodeEnvManifests(new)
	
	return !manifests.IsEqual(oldManifests, newManifests)
}

// Execute：核心执行逻辑
func (e *EnsureNodesEnv) Execute() (ctrl.Result, error) {
	ctx, _, bkeCluster, _, log := e.Ctx.Untie()
	
	// Step 1: 加载组件版本清单
	log.Info("Loading component manifests for NodeEnv", "version", bkeCluster.Spec.ClusterConfig.Cluster.KubernetesVersion)
	
	nodeEnvManifest, err := e.componentManager.GetComponentVersionManifest(ctx, 
		component.NodeEnvComponent, 
		bkeCluster.Spec.ClusterConfig.Cluster.KubernetesVersion,
	)
	if err != nil {
		return ctrl.Result{}, fmt.Errorf("load component manifest: %w", err)
	}
	
	// Step 2: 检查前置条件（兼容性、依赖等）
	if err := e.validatePrerequisites(ctx, bkeCluster, nodeEnvManifest); err != nil {
		return ctrl.Result{}, fmt.Errorf("prerequisite check failed: %w", err)
	}
	
	// Step 3: 比较版本并生成任务序列
	nodes, err := e.Ctx.NodeFetcher().GetBKENodesWrapperForCluster(ctx, bkeCluster)
	if err != nil {
		return ctrl.Result{}, fmt.Errorf("get nodes: %w", err)
	}
	
	tasks, err := e.componentManager.GenerateUpgradeTasks(
		bkeCluster,
		nodes,
		nodeEnvManifest,
	)
	if err != nil {
		return ctrl.Result{}, fmt.Errorf("generate upgrade tasks: %w", err)
	}
	
	// Step 4: 执行任务序列
	if err := e.executeTasks(ctx, tasks, bkeCluster, log); err != nil {
		return ctrl.Result{RequeueAfter: 10 * time.Second}, fmt.Errorf("execute tasks: %w", err)
	}
	
	// Step 5: 校验并更新状态
	if err := e.verifyAndUpdateStatus(ctx, bkeCluster, nodeEnvManifest, log); err != nil {
		return ctrl.Result{RequeueAfter: 10 * time.Second}, fmt.Errorf("verify and update status: %w", err)
	}
	
	log.Info("EnsureNodesEnv completed successfully")
	return ctrl.Result{}, nil
}

// 校验前置条件
func (e *EnsureNodesEnv) validatePrerequisites(
	ctx context.Context,
	cluster *bkev1beta1.BKECluster,
	manifest *component.LifecycleManifest,
) error {
	// 检查兼容性
	for _, dep := range manifest.ComponentVersion.Compatibility.Dependencies {
		otherManifest, err := e.componentManager.GetComponentVersionManifest(ctx,
			dep.Component,
			cluster.Spec.ClusterConfig.Cluster.KubernetesVersion,
		)
		if err != nil {
			return fmt.Errorf("dependency %s not found: %w", dep.Component, err)
		}
		
		if !isVersionInRange(otherManifest.ComponentVersion.Version, dep.MinVersion, dep.MaxVersion) {
			return fmt.Errorf("incompatible %s version: %s (requires %s-%s)",
				dep.Component, otherManifest.ComponentVersion.Version, dep.MinVersion, dep.MaxVersion)
		}
	}
	
	return nil
}

// 执行任务序列
func (e *EnsureNodesEnv) executeTasks(
	ctx context.Context,
	tasks []*component.Task,
	cluster *bkev1beta1.BKECluster,
	log logr.Logger,
) error {
	for _, task := range tasks {
		log.Info("Executing task", "name", task.Name, "type", task.Type)
		
		// 为每个节点执行任务
		taskCtx := &TaskExecutionContext{
			Cluster:          cluster,
			ComponentVersion: task.ComponentVersion,
			Nodes:            task.TargetNodes,
			Env:              task.Env,
		}
		
		// 调用通用任务执行器
		result, err := e.Ctx.TaskExecutor().Execute(ctx, task, taskCtx)
		if err != nil {
			if task.OnFailure == "Fail" {
				return fmt.Errorf("task %s failed: %w", task.Name, err)
			}
			log.Error(err, "Task failed but continuing", "name", task.Name)
		}
		
		// 等待任务完成（如果是异步的）
		if result.IsAsync {
			if err := e.waitForTaskCompletion(ctx, result.TaskID, task.Timeout); err != nil {
				return fmt.Errorf("wait for task completion: %w", err)
			}
		}
	}
	
	return nil
}

// 校验并更新状态
func (e *EnsureNodesEnv) verifyAndUpdateStatus(
	ctx context.Context,
	cluster *bkev1beta1.BKECluster,
	manifest *component.LifecycleManifest,
	log logr.Logger,
) error {
	// 执行验证脚本（来自 manifest）
	verifySteps := manifest.Lifecycle.Verify
	
	for _, step := range verifySteps {
		log.Info("Running verification", "name", step.Name)
		
		result, err := e.executeVerificationStep(ctx, step, cluster)
		if err != nil {
			return fmt.Errorf("verification failed: %w", err)
		}
		
		if !result.Success {
			return fmt.Errorf("verification %s failed: %s", step.Name, result.ErrorMessage)
		}
	}
	
	// 更新 Condition
	conditions.ConditionMark(cluster, bkev1beta1.NodesEnvReadyCondition, "True")
	
	return nil
}

	nodes, _ := e.Ctx.NodeFetcher().GetBKENodesWrapperForCluster(ctx, cluster)

	for _, node := range nodes.Items {

			needUpgrade = true
			break
		}

	}

// Etcd 升级状态机
type EtcdUpgradeState string

const (
	EtcdStateIdle                EtcdUpgradeState = "Idle"
	EtcdStateBackupPreUpgrade    EtcdUpgradeState = "BackupPreUpgrade"
	EtcdStateCheckClusterHealth  EtcdUpgradeState = "CheckClusterHealth"
	EtcdStateUpgradingMember     EtcdUpgradeState = "UpgradingMember"
	EtcdStateVerifyMemberUpgrade EtcdUpgradeState = "VerifyMemberUpgrade"
	EtcdStateCheckClusterHealthAfterMember EtcdUpgradeState = "CheckClusterHealthAfterMember"
	EtcdStateAllMembersUpgraded  EtcdUpgradeState = "AllMembersUpgraded"
	EtcdStateVerifyFinalCluster  EtcdUpgradeState = "VerifyFinalCluster"
	EtcdStateUpgradeCompleted    EtcdUpgradeState = "UpgradeCompleted"
	EtcdStateRollingBack         EtcdUpgradeState = "RollingBack"
	EtcdStateFailed              EtcdUpgradeState = "Failed"
)

func (e *EnsureEtcd) Execute() (ctrl.Result, error) {
	ctx, _, cluster, _, log := e.Ctx.Untie()
	
	// 获取当前升级状态
	status := e.getEtcdUpgradeStatus(cluster)
	log.Info("etcd upgrade state", "state", status.State, "member", status.CurrentMember, "totalMembers", status.TotalMembers)
	
	// 状态机驱动
	nextState, err := e.handleState(ctx, status, cluster, log)
	if err != nil {
		// 失败，进入回滚状态
		if status.State != EtcdStateRollingBack && status.State != EtcdStateFailed {
			e.setEtcdUpgradeStatus(cluster, EtcdStateRollingBack, err.Error())
			return ctrl.Result{RequeueAfter: 5 * time.Second}, nil
		}
	}
	
	// 更新状态
	if nextState != status.State {
		e.setEtcdUpgradeStatus(cluster, nextState, "")
		return ctrl.Result{RequeueAfter: 10 * time.Second}, nil
	}
	
	return ctrl.Result{}, nil
}

func (e *EnsureEtcd) handleState(
	ctx context.Context,
	status *EtcdUpgradeStatus,
	cluster *bkev1beta1.BKECluster,
	log logr.Logger,
) (EtcdUpgradeState, error) {
	switch status.State {
	case EtcdStateIdle:
		return e.handleStateIdle(ctx, status, cluster, log)
	
	case EtcdStateBackupPreUpgrade:
		return e.handleStateBackupPreUpgrade(ctx, status, cluster, log)
	
	case EtcdStateCheckClusterHealth:
		return e.handleStateCheckClusterHealth(ctx, status, cluster, log)
	
	case EtcdStateUpgradingMember:
		return e.handleStateUpgradingMember(ctx, status, cluster, log)
	
	case EtcdStateVerifyMemberUpgrade:
		return e.handleStateVerifyMemberUpgrade(ctx, status, cluster, log)
	
	case EtcdStateCheckClusterHealthAfterMember:
		return e.handleStateCheckClusterHealthAfterMember(ctx, status, cluster, log)
	
	case EtcdStateAllMembersUpgraded:
		return e.handleStateAllMembersUpgraded(ctx, status, cluster, log)
	
	case EtcdStateVerifyFinalCluster:
		return e.handleStateVerifyFinalCluster(ctx, status, cluster, log)
	
	case EtcdStateUpgradeCompleted:
		return EtcdStateUpgradeCompleted, nil
	
	case EtcdStateRollingBack:
		return e.handleStateRollingBack(ctx, status, cluster, log)
	
	case EtcdStateFailed:
		return EtcdStateFailed, nil
	
	default:
		return "", fmt.Errorf("unknown state: %s", status.State)

// State: Idle → 检查版本
func (e *EnsureEtcd) handleStateIdle(
	ctx context.Context,
	status *EtcdUpgradeStatus,
	cluster *bkev1beta1.BKECluster,
	log logr.Logger,
) (EtcdUpgradeState, error) {
	log.Info("Checking if etcd upgrade is needed")
	
	etcdVersion := cluster.Spec.ClusterConfig.Cluster.EtcdVersion
	currentVersion := e.getCurrentEtcdVersion(cluster)
	
	if etcdVersion == currentVersion {
		log.Info("etcd already at desired version", "version", etcdVersion)
		return EtcdStateUpgradeCompleted, nil
	}
	
	log.Info("etcd upgrade needed", "from", currentVersion, "to", etcdVersion)
	
	// 初始化升级状态
	status.SourceVersion = currentVersion
	status.TargetVersion = etcdVersion
	status.BackupPath = fmt.Sprintf("/opt/etcd-backup/pre-upgrade-%s", time.Now().Format("20060102-150405"))
	status.CurrentMember = 0
	status.TotalMembers = 0
	
	return EtcdStateBackupPreUpgrade, nil
}

// State: BackupPreUpgrade → 快照备份
func (e *EnsureEtcd) handleStateBackupPreUpgrade(
	ctx context.Context,
	status *EtcdUpgradeStatus,
	cluster *bkev1beta1.BKECluster,
	log logr.Logger,
) (EtcdUpgradeState, error) {
	log.Info("Creating etcd snapshot backup", "path", status.BackupPath)
	
	manifest, _ := e.componentManager.GetComponentVersionManifest(ctx, "etcd", status.TargetVersion)
	
	// 获取快照步骤
	backupStep := manifest.Lifecycle.Upgrade.GetStep("snapshot-backup")
	
	// 在主 etcd 节点执行备份
	task := &component.Task{
		Name: "backup-etcd",
		Type: backupStep.Type,
		Script: backupStep.Script,
		Env: map[string]string{
			"BACKUP_PATH": status.BackupPath,
		},
	}
	
	if result, err := e.Ctx.TaskExecutor().Execute(ctx, task, nil); err != nil {
		return EtcdStateBackupPreUpgrade, err
	}
	
	status.BackupPath = result.Output["backup_path"]
	log.Info("etcd snapshot created", "path", status.BackupPath)
	
	return EtcdStateCheckClusterHealth, nil
}

// State: CheckClusterHealth → 检查集群健康
func (e *EnsureEtcd) handleStateCheckClusterHealth(
	ctx context.Context,
	status *EtcdUpgradeStatus,
	cluster *bkev1beta1.BKECluster,
	log logr.Logger,
) (EtcdUpgradeState, error) {
	log.Info("Checking etcd cluster health before upgrade")
	
	health, members, err := e.checkEtcdHealth(ctx, cluster)
	if err != nil {
		return EtcdStateCheckClusterHealth, fmt.Errorf("etcd health check failed: %w", err)
	}
	
	if !health.IsHealthy {
		return EtcdStateCheckClusterHealth, fmt.Errorf("etcd cluster not healthy: %v", health.UnhealthyMembers)
	}
	
	status.TotalMembers = len(members)
	status.MemberStatus = make(map[int]string, len(members))
	for i, member := range members {
		status.MemberStatus[i] = "Ready"
	}
	
	log.Info("etcd cluster is healthy", "members", len(members))
	
	return EtcdStateUpgradingMember, nil
}

// State: UpgradingMember → 升级单个成员
func (e *EnsureEtcd) handleStateUpgradingMember(
	ctx context.Context,
	status *EtcdUpgradeStatus,
	cluster *bkev1beta1.BKECluster,
	log logr.Logger,
) (EtcdUpgradeState, error) {
	if status.CurrentMember >= status.TotalMembers {
		return EtcdStateAllMembersUpgraded, nil
	}
	
	memberIndex := status.CurrentMember
	log.Info("Upgrading etcd member", "index", memberIndex, "total", status.TotalMembers)
	
	manifest, _ := e.componentManager.GetComponentVersionManifest(ctx, "etcd", status.TargetVersion)
	
	// 获取升级步骤
	upgradeMemberStep := manifest.Lifecycle.Upgrade.GetStep("rolling-upgrade")
	
	// 让该成员让出 leader
	e.drainEtcdLeadership(ctx, memberIndex, log)
	
	// 停止 etcd 进程
	// 备份数据（已在全局备份中完成）
	// 升级二进制
	// 启动新版本
	
	task := &component.Task{
		Name: "upgrade-etcd-member",
		Type: upgradeMemberStep.Type,
		Params: upgradeMemberStep.Params,
		Env: map[string]string{
			"MEMBER_INDEX":      fmt.Sprintf("%d", memberIndex),
			"SOURCE_VERSION":    status.SourceVersion,
			"TARGET_VERSION":    status.TargetVersion,
		},
	}
	
	if _, err := e.Ctx.TaskExecutor().Execute(ctx, task, nil); err != nil {
		log.Error(err, "Failed to upgrade etcd member", "index", memberIndex)
		status.MemberStatus[memberIndex] = "UpgradeFailed"
		return EtcdStateUpgradingMember, err
	}
	
	status.MemberStatus[memberIndex] = "UpgradeInProgress"
	
	return EtcdStateVerifyMemberUpgrade, nil
}

// State: VerifyMemberUpgrade → 验证成员升级
func (e *EnsureEtcd) handleStateVerifyMemberUpgrade(
	ctx context.Context,
	status *EtcdUpgradeStatus,
	cluster *bkev1beta1.BKECluster,
	log logr.Logger,
) (EtcdUpgradeState, error) {
	memberIndex := status.CurrentMember
	log.Info("Verifying etcd member upgrade", "index", memberIndex)
	
	// 检查成员是否已升级
	memberVersion, err := e.getEtcdMemberVersion(ctx, cluster, memberIndex)
	if err != nil {
		return EtcdStateVerifyMemberUpgrade, err
	}
	
	if memberVersion != status.TargetVersion {
		log.Info("Member not yet upgraded", "index", memberIndex, "version", memberVersion)
		return EtcdStateVerifyMemberUpgrade, nil  // 重试
	}
	
	log.Info("Member upgraded successfully", "index", memberIndex, "version", memberVersion)
	status.MemberStatus[memberIndex] = "Upgraded"
	status.CurrentMember++
	
	return EtcdStateCheckClusterHealthAfterMember, nil
}

// State: CheckClusterHealthAfterMember → 升级后检查集群
func (e *EnsureEtcd) handleStateCheckClusterHealthAfterMember(
	ctx context.Context,
	status *EtcdUpgradeStatus,
	cluster *bkev1beta1.BKECluster,
	log logr.Logger,
) (EtcdUpgradeState, error) {
	log.Info("Checking etcd cluster health after member upgrade", "member", status.CurrentMember-1)
	
	health, _, err := e.checkEtcdHealth(ctx, cluster)
	if err != nil {
		log.Error(err, "Health check failed, retrying")
		return EtcdStateCheckClusterHealthAfterMember, nil  // 重试
	}
	
	if !health.IsHealthy {
		return EtcdStateCheckClusterHealthAfterMember, fmt.Errorf("etcd cluster unhealthy after member upgrade")
	}
	
	log.Info("etcd cluster is healthy after member upgrade")
	
	// 检查是否所有成员都已升级
	if status.CurrentMember >= status.TotalMembers {
		return EtcdStateAllMembersUpgraded, nil
	}
	
	return EtcdStateUpgradingMember, nil
}

// State: AllMembersUpgraded → 所有成员已升级
func (e *EnsureEtcd) handleStateAllMembersUpgraded(

	cluster *bkev1beta1.BKECluster,
	log logr.Logger,

// State: VerifyFinalCluster → 最终集群验证
func (e *EnsureEtcd) handleStateVerifyFinalCluster(
	ctx context.Context,
	status *EtcdUpgradeStatus,
	cluster *bkev1beta1.BKECluster,
	log logr.Logger,
) (EtcdUpgradeState, error) {
	log.Info("Verifying final etcd cluster state")
	
	manifest, _ := e.componentManager.GetComponentVersionManifest(ctx, "etcd", status.TargetVersion)
	
	// 执行最终验证步骤
	verifyStep := manifest.Lifecycle.Upgrade.GetStep("post-upgrade-verify")
	
	task := &component.Task{
		Name: "verify-etcd-upgrade",
		Type: verifyStep.Type,
		Script: verifyStep.Script,
	}
	
	if _, err := e.Ctx.TaskExecutor().Execute(ctx, task, nil); err != nil {
		log.Error(err, "Final verification failed")
		return EtcdStateVerifyFinalCluster, err
	}
	
	log.Info("etcd cluster verification passed")
	
	return EtcdStateUpgradeCompleted, nil
}

// State: RollingBack → 回滚升级
func (e *EnsureEtcd) handleStateRollingBack(
	ctx context.Context,
	status *EtcdUpgradeStatus,
	cluster *bkev1beta1.BKECluster,
	log logr.Logger,
) (EtcdUpgradeState, error) {
	log.Error(nil, "Rolling back etcd upgrade", "reason", status.Error)

		log.Error(err, "Snapshot restore failed")
		return EtcdStateFailed, err
	}
	
	log.Info("etcd restored from snapshot")
	
	return EtcdStateFailed, fmt.Errorf("upgrade failed and rolled back: %s", status.Error)
}

## 六、完整设计思路总结
### 6.1 分层模型
```
┌─────────────────────────────────────────────────────────────────────────┐
│                         Application Layer                                 │
│  BKECluster Controller → Phase Executor → Task Orchestrator              │
├─────────────────────────────────────────────────────────────────────────┤
│                      Component Layer                                       │
│  ComponentVersion Manifest ← → Component Manager ← → Compatibility Check  │
├─────────────────────────────────────────────────────────────────────────┤
│                        Execution Layer                                     │
│  Task Executor → {Builtin Plugins, Shell Scripts, K8s Resources}         │
├─────────────────────────────────────────────────────────────────────────┤
│                      Infrastructure Layer                                  │
│  BKE Agent ← → Nodes (OS, Binaries, Services)                           │
└─────────────────────────────────────────────────────────────────────────┘
```
### 6.2 核心原则
1. **声明式优于命令式**
   - 所有安装/卸载/升级逻辑声明在 YAML 中
   - 控制器只负责驱动状态机
2. **版本管理最优先**
   - 每个组件独立版本化
   - 版本对比 → 生成差异任务
3. **容错与回滚内置**
   - 每一步都有失败处理规则
   - 支持自动回滚
4. **状态机驱动有状态升级**
   - etcd 等有状态组件用状态机
   - 幂等操作，可重入
5. **兼容性前置校验**
   - 升级前检查依赖版本
   - 阻止不兼容升级

// 生成迁移清单
func GenerateMigrationManifest(
	ctx context.Context,
	legacyVersions map[string]string,
	targetVersions map[string]string,
) (*component.MigrationManifest, error) {
	manifest := &component.MigrationManifest{
		Steps: []*component.MigrationStep{},
	}
	
	// 逐个组件生成迁移步骤
	for component, legacyVersion := range legacyVersions {
		targetVersion := targetVersions[component]
		
		if legacyVersion != targetVersion {
			// 需要升级
			manifest.Steps = append(manifest.Steps, &component.MigrationStep{
				Type:       "upgrade",
				Component:  component,
				From:       legacyVersion,
				To:         targetVersion,
				Order:      1,  // 执行顺序
			})
		} else {
			// 仅需导入配置
			manifest.Steps = append(manifest.Steps, &component.MigrationStep{
				Type:       "import",
				Component:  component,
				Version:    legacyVersion,
				Order:      1,
			})
		}
	}
	
	return manifest, nil
}

// 执行迁移
func ExecuteMigration(
	ctx context.Context,
	manifest *component.MigrationManifest,
) error {
	for _, step := range manifest.Steps {
		fmt.Printf("Executing migration step: %s %s from %s to %s\n",
			step.Type, step.Component, step.From, step.To)
		
		switch step.Type {
		case "upgrade":
			// 调用升级逻辑
			if err := executeUpgrade(ctx, step); err != nil {
				return err
			}
		
		case "import":
			// 导入配置
			if err := importConfig(ctx, step); err != nil {
				return err
			}
		}
	}
	
	return nil
}

      status: "Incompatible"

  # 升级路径
  upgradePaths:
    - from: "kubernetes 1.27.0"

      supported: true
      strategy: "RollingUpgrade"

      downtime: 0  # 无停机升级
      notes: "Master 串行，worker 并行"

      to: "kubernetes 1.27.0"
      supported: false
      reason: "Downgrade not supported"

  dependencies:

      requires:
        - etcd: ">= 3.5.0, < 3.7.0"
        - containerd: ">= 1.6.0"
        - cni-plugin: ">= 1.2.0"

## 总结：完整设计要点

| 设计要点 | 说明 |
|---------|------|
| **声明式优于��令式** | 所有 LCM 逻辑在 Manifest 中，控制器只调和 |
| **版本即一等公民** | ComponentVersion CRD 管理所有版本信息 |
| **兼容性前置** | 升级前检查依赖版本，阻止不兼容操作 |
| **有状态支持** | 状态机驱动 etcd 等有状态组件升级 |
| **容错与回滚** | 每一步都有失败处理和自动回滚 |
| **幂等操作** | 所有步骤可重入，支持故障恢复 |
| **模块化任务执行** | shell、builtin、k8s 三种执行方式 |
| **渐进式迁移** | 旧系统自动检测与导入，无缝升级 |
| **可观测性** | 完整的状态机日志和事件 |
| **工作量 16 周** | 2-3 人团队完成整改 |

这个方案通过分层架构、声明式定义、版本管理、有状态支持和完整容错机制，实现了 BKE 从命令式到声明式、从手工运维到自动化升级的转变，大大提升了集群管理的可靠性和可维护性。
