# SIG-Installation：openFuyao 集群安装与版本生命周期管理

## 使命与愿景

SIG-Installation 负责 openFuyao 发行版集群的**全生命周期管理**——从集群创建、节点扩缩容、版本升级到集群销毁，提供声明式、可观测、生产级的安装与升级能力。我们的愿景是让 openFuyao 的安装升级体验达到公有云托管集群的水平：**一键声明版本，系统自动完成兼容性校验、依赖编排与滚动升级**。

## 技术目标

| # | 目标 | 当前状态 | 目标状态 |
|---|------|---------|---------|
| T1 | **声明式版本管理** | 版本信息散落在 BKECluster.Spec 各字段，无统一版本入口 | 引入 ClusterVersion / ReleaseImage / UpgradePath CRD，版本声明与集群实例解耦 |
| T2 | **DAG 驱动的升级编排** | Phase 执行顺序硬编码（27 个 Phase 线性排列），无法并行、回滚成本高 | 基于 ComponentVersion 依赖拓扑自动构建安装/升级 DAG，支持并行、失败策略、回滚 |
| T3 | **OCI 原生的版本分发** | 版本清单通过 ConfigMap 手动分发，离线环境适配困难 | ReleaseImage / UpgradePath 通过 OCI 镜像分发，支持 Registry Mirror 与断网降级 |
| T4 | **组件化与可扩展** | 新增组件需侵入核心控制器代码，耦合度高 | ComponentFactory 注册机制 + YAML 清单驱动，新组件零代码侵入 |
| T5 | **升级路径与兼容性** | 组件间版本依赖无集中校验，易出现不兼容 | UpgradePath 图引擎 + 兼容性约束求解，升级前自动拦截不兼容路径 |
| T6 | **节点侧 Agent 演进** | bkeagent 17 个内置插件硬编码，新增指令需重新编译 | 插件注册表 + 动态加载机制，支持第三方扩展 |

## 技术方案

### 方案一：声明式版本管理（ClusterVersion / ReleaseImage / UpgradePath）

引入三个核心 CRD，建立"集群实例 → 版本声明 → 版本清单 → 升级路径"的完整链路：

- **ClusterVersion**：1:1 关联 BKECluster，声明 `desiredVersion` 与 `currentVersion`，是升级的统一入口
- **ReleaseImage**：不可变版本清单，通过 OCI 镜像分发，定义 install/upgrade 组件列表
- **UpgradePath**：聚合所有合法升级路径边，支持 blocked / deprecated / preCheck / postCheck

### 方案二：DAG 调度引擎

从 ReleaseImage 解析 ComponentVersion 列表，根据 `dependencies` 字段构建有向无环图（DAG），按拓扑序调度 Phase 执行：

- **安装 DAG**：从零构建集群，按依赖拓扑编排
- **升级 DAG**：基于当前版本与目标版本的差异集构建，仅执行需变更的组件
- **失败策略**：FailFast（默认）/ Continue / Rollback，每个组件可独立配置

### 方案三：ComponentFactory + YAML 清单驱动

- **YAML 清单模式**：EnsureProviderSelfUpgrade / EnsureAgentUpgrade / EnsureComponentUpgrade 等转为纯 YAML 声明
- **Inline 模式**：EnsureEtcdUpgrade / EnsureMasterUpgrade 等保留 Go 实现，通过 `ComponentFactory.Register()` 注册
- **NeedExecute() 复用**：严禁新增 ShouldUpgrade() 接口，通过版本上下文注入增强现有 NeedExecute()

### 方案四：OCI 原生分发与离线适配

- ReleaseImage 与 UpgradePath 打包为 OCI 镜像，通过 `oras` / `skopeo` 分发
- 支持 Registry Mirror，适配离线/气隙环境
- UpgradePath Digest Monitor 持续监控 `:latest` 镜像变更，自动同步升级路径

## 贡献方向与合作伙伴匹配

| 技术方向 | 所需技能 | 涉及层级 | 复杂度 | 适合角色 |

| 🔴 **CRD 设计与 API 治理** | Kubernetes API 设计、kubebuilder | 管理集群 | ⭐⭐⭐ | 平台架构师 |
| 🔴 **DAG 调度引擎** | 图算法、拓扑排序、并发调度 | 管理集群 | ⭐⭐⭐⭐ | 调度系统工程师 |
| 🔴 **UpgradePath 图引擎** | 图论、约束求解、SemVer | 管理集群 | ⭐⭐⭐⭐ | 版本管理专家 |
| 🟡 **OCI 分发与离线适配** | OCI 规范、oras/skopeo | 引导集群 + 管理集群 | ⭐⭐⭐ | 容器镜像工程师 |
| 🟡 **ComponentFactory 注册机制** | Go 接口设计、插件模式 | 管理集群 | ⭐⭐ | Go 后端工程师 |
| 🟡 **BKEAgent 插件化** | Go 插件机制、接口抽象 | 业务集群 | ⭐⭐ | Agent 开发工程师 |
| 🟢 **YAML 清单引擎** | K8s Manifest、Kustomize | 管理集群 | ⭐⭐ | K8s 运维工程师 |
| 🟢 **升级可观测性** | Prometheus、Event、Condition | 管理集群 + 业务集群 | ⭐⭐ | 可观测性工程师 |
