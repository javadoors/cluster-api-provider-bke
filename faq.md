## 总结

以上架构图展示了openFuyao平台各组件的详细架构和相互关系：

1. **cluster-api-provider-bke**: 核心控制器，负责集群生命周期管理
2. **bkeadm**: CLI工具，用于初始化和部署
3. **bke-manifests**: Manifest仓库，提供所有组件的YAML模板
4. **installer-service**: 安装服务后端，提供集群管理API
5. **installer-website**: 安装服务前端，提供安装向导界面
6. **bke-console-service**: 控制台后端，提供认证、路由、代理功能
7. **bke-console-website**: 控制台前端，提供用户管理界面
8. **openfuyao-system-controller**: 系统控制器，负责平台级管理

这些组件协同工作，共同构成了一个完整的Kubernetes集群管理平台。

| 维度 | bke-manifests | openfuyao-system-controller |

## 详细说明
### bke-manifests
**本质**：静态资源包
```
┌─────────────────────────────────────────────────┐
│              bke-manifests 镜像                  │
├─────────────────────────────────────────────────┤
│  kubernetes/                                    │
│   ├── calico/v3.25.0/calico.yaml               │
│   ├── coredns/v1.10.1/coredns.yaml             │
│   ├── prometheus/...                           │
│   ├── cluster-api/...                          │
│   └── ... (各种addon清单)                       │
│  bkeinitscript/                                 │
│   └── ... (初始化脚本)                          │
└─────────────────────────────────────────────────┘
```
- 存储**各版本**的Kubernetes组件部署清单
- 使用Go template进行镜像地址参数化 (`{{.repo}}image:tag`)
- 被 [cluster-api-provider-bke](file:///d:/code/github/cluster-api-provider-bke) 读取并部署到工作负载集群
- [main.go](file:///d:/code/github/bke-manifests/main.go) 只是一个信号处理程序，保持容器运行

### openfuyao-system-controller
**本质**：安装器/部署工具
```
┌─────────────────────────────────────────────────┐
│         openfuyao-system-controller             │
├─────────────────────────────────────────────────┤
│  install.sh  ─────► 安装流程编排                 │
│     ├── install_installer_service()            │
│     ├── install_console_service()              │
│     ├── install_kube_prometheus()              │
│     └── ...                                     │
│  resource/                                      │
│   ├── kube-prometheus/  (监控组件)              │
│   ├── metrics-server/                          │
│   ├── ingress-nginx/                           │
│   └── oauth-webhook/                           │
└─────────────────────────────────────────────────┘
```
- 在**管理集群**上部署openFuyao管理面组件
- 通过Helm charts安装：installer-service、console-service、console-website、monitoring-service等
- 包含完整的安装逻辑：下载charts、创建证书、配置TLS、安装组件
- 作为Deployment运行，initContainer执行安装，主容器保持运行

| 组件 | 作用对象 | 功能 |
|------|---------|------|
| **bke-manifests** | 工作负载集群 | 提供**addon部署清单**，由cluster-api-provider-bke读取并部署 |

简单来说：**bke-manifests是"材料库"，openfuyao-system-controller是"安装工"**。
